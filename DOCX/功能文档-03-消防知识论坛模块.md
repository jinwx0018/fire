# 功能文档 - 模块三：消防知识论坛模块

本文档对应需求与设计中的**消防知识论坛模块**，说明帖子发布与审核、评论互动、**点赞（Redis Set）**的代码设计与实现。

---

## 一、功能概述

用户可发布帖子（标题、内容），帖子状态支持 1 正常、0 待审核、-1 违规；管理员可审核通过或驳回并填写驳回理由。用户可对帖子评论（支持一级与 parent_id 回复）、对帖子点赞/取消点赞。点赞使用 **Redis Set** 存储每帖的点赞用户 ID 集合，避免重复点赞，点赞数可同步回数据库用于展示。

---

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 帖子发布 | 用户提交标题、内容，更新数据库 | 新建 forum_post，status 默认 0（待审核）或按业务设为 1；user_id 为当前用户 |
| 评论互动 | 记录评论者、帖子 ID、内容；可回复（parent_id） | forum_comment 表：post_id、user_id、parent_id、content；发表评论后帖子 comment_count +1 |
| 点赞功能 | Redis Set 存点赞用户 ID，避免重复；点赞数实时更新 | Key：forum:post:like:{postId}，Value：Set 存 userId 字符串；点赞/取消时更新 Set 并 size 写回 forum_post.like_count |
| 帖子审核 | 管理员通过或驳回，更新状态与驳回理由 | status 1 通过、-1 驳回；reject_reason 存驳回理由 |

## 四、数据库设计

脚本：`DB/03_forum.sql`。

| 表名 | 说明 |
|------|------|
| forum_post | id, user_id, title, content, status(1 正常 0 待审核 -1 违规), reject_reason, view_count, like_count, comment_count, deleted, create_time, update_time；FULLTEXT(title, content) |
| forum_comment | id, post_id, user_id, parent_id, content, deleted, create_time, update_time |

**Redis**：Key = `forum:post:like:{postId}`，类型 Set，成员为 userId 的字符串；点赞时 SADD，取消时 SREM；SCARD 得到点赞数并回写 DB。

---

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | ForumController | 帖子列表/详情/发布/修改/删除、点赞、评论列表/发表/删除 |
| Controller | AdminForumController | 帖子审核 audit |
| Service | ForumService、ForumServiceImpl | 依赖 ForumPostMapper、ForumCommentMapper、SysUserMapper、StringRedisTemplate |
| Entity | ForumPost、ForumComment | @TableLogic 对应 deleted |

### 5.2 点赞实现（Redis Set）

- **ForumServiceImpl.likePost(postId, userId)**：  
  - Key = `forum:post:like:` + postId。  
  - 使用 `redisTemplate.opsForSet().isMember(key, String.valueOf(userId))` 判断是否已点赞。  
  - 若已点赞：SREM 移除该 userId，liked=false；若未点赞：SADD 添加，liked=true。  
  - 使用 `redisTemplate.opsForSet().size(key)` 得到当前点赞数，通过 LambdaUpdateWrapper 更新 forum_post 的 like_count 字段，保证列表/详情展示一致。  
  - 返回 Map：liked、likeCount。

### 5.3 帖子详情与 liked

- **postDetail(postId, userId)**：查 forum_post，转为 Map（含用户信息通过 SysUserMapper）。若 userId 非空，用 Redis Set isMember 判断该用户是否已点赞，设置 map.put("liked", true/false)。

### 5.4 评论与计数

- **addComment**：插入 forum_comment；再用 LambdaUpdateWrapper 对 forum_post 执行 `setSql("comment_count = comment_count + 1")`。  
- **commentList**：按 post_id、deleted=0 分页查 forum_comment，每条附带 user 的 userName、avatar。

### 5.5 审核

- **auditPost(postId, status, rejectReason)**：根据 postId 查 ForumPost，setStatus(status)、setRejectReason(rejectReason)，updateById。status 传 1 为通过，-1 为驳回。

---

## 六、注意事项

- 帖子列表用户端通常只查 status=1；管理端可传 status 筛选待审核(0)或违规(-1)。  
- 点赞数以 Redis 为准，回写 DB 便于列表排序与持久化；若 Redis 清空，可依赖 DB 的 like_count 做兜底（需考虑与 Redis 的同步策略）。
