# 功能文档 - 模块三：消防知识论坛模块

本文档对应需求与设计中的**消防知识论坛模块**，说明帖子发布与审核、评论互动、**点赞（Redis Set）**的代码设计与实现，以及**公开可见性与审核安全**约束。本文档与当前后端/管理端实现保持一致。

---

## 一、功能概述

用户可发布帖子（标题、内容），帖子状态支持 **1 已通过**、**0 待审核**、**-1 已驳回**；管理员可审核通过或驳回，**驳回须填写驳回理由**。用户可对已通过审核的帖子评论、点赞/取消点赞。点赞使用 **Redis Set** 存储每帖的点赞用户 ID 集合；**未配置 Redis** 时点赞不持久化「谁点过赞」，接口行为见实现说明。

**作者修改帖子**后，系统将 **status 置为 0（待审核）** 并 **清空 reject_reason**，需管理员再次审核通过后才会出现在公开列表。

---

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 帖子发布 | 用户提交标题、内容 | 新建 forum_post，**status 默认 0（待审核）** |
| 公开列表安全 | 用户端列表不得出现未审核帖 | **`GET /forum/post/list` 固定仅 status=1**，不接受 status 参数 |
| 管理端列表 | 审核台按状态筛选 | **`GET /admin/forum/post/list`**（管理员），支持 status、keyword |
| 详情与评论可见性 | 未通过帖不得被匿名爬取 | **已通过**：任何人可看详情与评论；**待审核/驳回**：仅 **作者本人** 或 **管理员**（JWT）可看详情与评论列表，否则 **404** |
| 浏览量 | 详情被合法查看时统计 | **`GET /forum/post/{id}`** 在通过 `assertCanViewPost` 后 **`view_count + 1`** |
| 作者改帖 | 修改后重新进入审核 | **`PUT /forum/post/{id}`**（本人）：更新标题/正文后 **status→0**，**reject_reason 清空** |
| 互动约束 | 未通过帖不应被点赞、评论 | **点赞、评论**前校验 **status=1**，否则业务错误；评论 content 非空；**postId、parentId** 做格式与业务校验 |
| 评论互动 | 一级与 parent_id 回复 | forum_comment；通过后 **comment_count +1**；**parent_id** 须存在且 **属于同一 post_id** |
| 删评计数 | 软删与统计一致 | 用户/管理端删评（软删）后 **`comment_count` 减 1**（不小于 0，MySQL `GREATEST`） |
| 点赞功能 | Redis Set + DB like_count | Key：`forum:post:like:{postId}`；**点赞成功**写入推荐行为 **LIKE + FORUM_POST** |
| 评论行为 | 与推荐联动 | **发表评论**后写入 **COMMENT + FORUM_POST** |
| 帖子审核 | 通过/驳回 | `PUT /admin/forum/post/audit/{id}`；**status 仅允许 1 / -1**；**驳回时 reject_reason 必填**；通过时清空 reject_reason |
| 管理处置 | 违规内容删除 | **`DELETE /admin/forum/post/{id}`** 软删帖子；**`DELETE /admin/forum/comment/{id}`** 软删评论并扣减 **comment_count** |

## 三、接口列表

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | /forum/post/list | 帖子分页，**仅已通过**；keyword 可选 | 否 |
| GET | /admin/forum/post/list | **管理端**帖子分页，支持 **status**、keyword | 管理员 |
| GET | /forum/post/{id} | 帖子详情（含浏览量递增）；未通过帖仅作者/管理员 | 可选 |
| POST | /forum/post | 发布帖子 | 是 |
| PUT | /forum/post/{id} | 修改帖子（本人）；保存后 **status→0** 待重审 | 是 |
| DELETE | /forum/post/{id} | 删除帖子（本人，软删） | 是 |
| POST | /forum/post/like/{postId} | 点赞切换；**仅已通过帖** | 是 |
| GET | /forum/comment/list | 评论分页；**可见性与帖子一致** | 可选（见第六节） |
| POST | /forum/comment | 发表评论；**仅已通过帖**；body：`postId`、`content`、可选 `parentId` | 是 |
| DELETE | /forum/comment/{id} | 删除评论（本人，软删）；**comment_count -1** | 是 |
| PUT | /admin/forum/post/audit/{id} | 审核；body：`status`（1/-1）、驳回时 **`rejectReason` 必填** | 管理员 |
| DELETE | /admin/forum/post/{id} | 删除帖子（软删） | 管理员 |
| DELETE | /admin/forum/comment/{id} | 删除评论（软删）；**comment_count -1** | 管理员 |

## 四、数据库设计

脚本：`DB/03_forum.sql`。

| 表名 | 说明 |
|------|------|
| forum_post | id, user_id, title, content, status, reject_reason, view_count, like_count, comment_count, deleted, create_time, update_time |
| forum_comment | id, post_id, user_id, parent_id, content, deleted, create_time, update_time |

**Redis**：Key = `forum:post:like:{postId}`，类型 Set；点赞数回写 forum_post.like_count。

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | ForumController | 用户端论坛 API（列表、详情、发帖、改帖、删帖、点赞、评论、删评） |
| Controller | AdminForumController | **帖子分页（审核台）**、**审核**、**管理端删帖/删评** |
| Service | ForumService、ForumServiceImpl | **adminPostList**、**adminDeletePost**、**adminDeleteComment**、**assertCanViewPost**、**assertApprovedForInteraction**、**auditPost**（含参数校验）；依赖 **SysRoleMapper** 判断管理员 |
| Service | RecommendService | 点赞/评论后 **recordBehavior** |
| 配置 | WebMvcConfig | **`/forum/comment/list` 不在 JWT 全排除列表**，须经过拦截器 |
| 配置 | JwtInterceptor | **`/forum/comment/list` 为可选认证路径**：无 Token 放行且 `userId=null`；有 Token 则解析 **userId**，供作者/管理员查看未通过帖的评论 |

### 5.2 点赞实现（Redis Set）

见实现：`ForumServiceImpl.likePost`；互动前调用 **assertApprovedForInteraction**。无 Redis 时接口返回当前 DB 点赞数且不切换点赞状态（实现以代码为准）。

### 5.3 帖子详情与评论列表

- **postDetail**：`assertCanViewPost` 后 **`view_count` 自增**，再查库组装 Map；含 **rejectReason**（作者/管理员可见驳回原因）。  
- **commentList(postId, …, viewerUserId)**：先加载帖子并 **assertCanViewPost**；与详情可见性一致。

### 5.4 审核与改帖

- **auditPost**：校验 **status ∈ {1, -1}**；**status = -1 时 reject_reason 非空**；**status = 1** 时可将 reject_reason 置空。  
- **updatePost**：校验作者身份；更新内容后 **status = 0**、**reject_reason = null**。

### 5.5 删除与计数

- **deleteComment**（用户）：校验评论作者；软删后 **帖子 comment_count 减 1**（下限 0）。  
- **adminDeleteComment**：管理员软删评论，同样扣减 **comment_count**。  
- **deletePost / adminDeletePost**：帖子软删（**@TableLogic**）。

## 六、注意事项

- 管理端前端 **`getPostPage`** 须请求 **`/admin/forum/post/list`**，不得依赖用户端列表传 status。  
- **`GET /forum/comment/list`** 须在 **JwtInterceptor** 中按「可选认证」处理（见 5.1）：反代环境下作者/管理员带 **Authorization** 时才能解析 **userId**，从而查看**自己待审核/驳回帖**的评论；匿名访问已通过帖的评论时无需 Token。  
- 未配置 Redis 时，点赞与「是否已赞」以实现为准，部署生产环境建议启用 Redis 以保持与 **like_count** 一致。

## 七、管理端前端

- **帖子审核**：`admin/src/views/forum/PostAudit.vue`；API：`admin/src/api/forum.js`。  
  - **`getPostPage`** → **`/admin/forum/post/list`**（支持 status、keyword）。  
  - **`auditPost`** → **`PUT /admin/forum/post/audit/{id}`**（驳回时前端需采集 **rejectReason**，与后端必填一致）。  
  - **`deletePost`** → **`DELETE /admin/forum/post/{id}`**；**`deleteComment`** → **`DELETE /admin/forum/comment/{id}`**（弹窗中管理评论时使用）。

## 八、用户端前端（参考）

- 论坛列表、发帖、详情、评论与点赞：`client/src/views/forum/List.vue`、`Publish.vue`、`Detail.vue`；API：`client/src/api/forum.js`。  
- 路由 **`/forum/publish`** 须注册在 **`/forum/:id`** 之前，避免 `publish` 被当作帖子 ID。
