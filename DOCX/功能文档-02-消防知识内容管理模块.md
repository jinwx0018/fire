# 功能文档 - 模块二：消防知识内容管理模块

本文档对应需求与设计中的**消防知识内容管理模块**，说明内容发布与查询、分类管理、收藏、分享、点赞通知等能力的**代码设计与实现**。

---

## 一、功能概述

管理员或作者可发布、更新、删除消防知识内容；支持按分类、标题模糊搜索与分页查询；支持知识收藏、分享链接生成、点赞与作者获赞通知。分类由管理端维护（增删改）。

---

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 内容发布 | 管理员或作者更新内容保存到数据库 | POST /content 接收 title、categoryId、content、cover、tags、status；authorId 取当前登录用户，仅 AUTHOR/ADMIN 角色可调用 |
| 作者发布与审核 | 作者发布的内容需管理员审核通过后才在用户端展示 | 作者端通过 `GET /content/my/drafts` 管理“我的内容”（返回当前作者全部状态内容），草稿状态为 0，点击“提交审核”经 `POST /content/publish/{id}` 将状态置为待审核(3)；管理员在管理端使用 `GET /admin/content/page?status=3` 筛出待审核，结合 `PUT /admin/content/audit/{id}` 执行通过(status=1)/驳回(status=0)；**用户端公开列表** `GET /content/list` 仅已发布，不支持按状态筛待审核；审核通过后的内容原作者仍可在“我的内容”中继续编辑/删除 |
| 审核驳回原因 | 管理员驳回时需填写原因，作者可在“我的内容”和详情页查看 | knowledge_content 新增 reject_reason 字段记录最近一次驳回原因；管理端审核接口接收 rejectReason，service 中写入该字段；用户端草稿列表与详情接口返回并展示该字段 |
| 内容查询 | 按分类、标题、标签搜索与分页 | `GET /content/list` **仅返回已发布**（固定 status=1），支持 `categoryId`、`title`、`tag`、`sortBy(latest/hot)`；管理端全状态列表使用 `GET /admin/content/page` |
| 详情可见性（下架） | 下架内容不对访客公开 | `GET /content/{id}`：**仅已发布(1)对所有人可见**；**下架(2)、草稿、待审核**等仅 **作者或管理员** 可查，否则 **404**（与草稿/待审一致）；浏览量仅对已发布且非作者/非管理员的访问递增 |
| 内容管理 | 作者、管理员对内容增删改查 | 逻辑删除：MyBatis-Plus 配置 logic-delete；**update/delete** 在 Service 层校验 **作者本人或 ADMIN**；作者不能通过新建/修改接口直接将 status 置为已发布(1)，须走提交审核或管理员审核 |
| 生命周期管理 | 支持下架与恢复发布、回收站、彻底删除 | 管理端可将内容状态在 `1(发布)` 与 `2(下架)` 间切换；删除为逻辑删除进回收站；管理员可**恢复**、**彻底删除**单条或**清空回收站**（物理删除）；支持**批量逻辑删除**、**批量下架**、**批量恢复发布** |
| 管理端列表体验 | 状态筛选、待审核优先、与回收站分离 | 知识内容列表支持按状态筛选；进入页面时若存在待审核内容则默认筛「待审核」，否则默认「已发布」；回收站为独立菜单页，不与正常列表混排 |
| 下架通知 | 管理员下架已发布内容时通知作者 | `changeStatus` 为下架(2)且原状态为已发布、且操作者非作者时，向作者写入 `user_notification` |
| 富文本编辑 | 正文使用 wangEditor，支持插图 | 用户端/管理端表单使用富文本组件；图片经 `POST /content/editor/image` 或 `POST /admin/content/editor/image` 上传后插入正文 HTML；详情页继续安全渲染 |
| 分页体验 | 列表分页避免“无限空页” | 前后端统一使用 `total` 控制下一页按钮禁用；当当前页越界且返回空数据时自动回退上一页 |
| 收藏、分享、点赞与通知 | 用户可收藏知识并查看收藏列表、生成分享链接、点赞知识；作者接收内容获赞通知 | 收藏：user_collection 表唯一 (user_id, content_id)；分享：`GET /content/share/{contentId}` 返回分享链接；点赞：user_content_like 表唯一 (user_id, content_id)，同步维护 knowledge_content.like_count，并在非本人点赞时写入 user_notification；作者可在“我的内容”看到点赞数 |

## 四、数据库设计

核心脚本：`DB/02_knowledge.sql`（分类、内容、标签、收藏等）。

| 表名 | 说明 |
|------|------|
| knowledge_category | 分类：id, name, sort, status |
| knowledge_content | 内容：id, title, category_id, content, cover, summary, author_id, view_count, like_count, status(0 草稿 1 已发布 2 下架 3 待审核), reject_reason(最近一次审核驳回原因，仅作者可见), deleted(逻辑删除), create_time, update_time；表上建有 FULLTEXT(title)（**当前业务接口未单独暴露全文检索**，仅标题模糊查询走 LIKE） |
| knowledge_tag | 标签：id, name |
| knowledge_content_tag | 内容-标签多对多：content_id, tag_id |
| user_collection | 用户收藏：user_id, content_id，唯一约束防重复 |

**依赖脚本（与下架通知、点赞、审计相关）**

| 脚本 | 表名 | 说明 |
|------|------|------|
| `DB/10_user_security_and_audit.sql` | user_notification | 站内通知（管理员下架已发布内容通知作者、点赞通知作者等）；未执行时相关通知写入会跳过或失败，不影响主流程 |

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | ContentController | 用户端：list（仅已发布）、detail、save、update、delete、collect、uncollect、collectList、share、like、categoryList、myDrafts、publish、`POST /content/editor/image` 插图上传 |
| Controller | AdminContentCategoryController | 管理端：知识分页、审核、状态、回收站恢复/彻底删除/清空、批量删除/下架/恢复发布、富文本插图、分类 CRUD |
| Controller | UserNotificationController | 用户端：站内通知分页、未读数、详情、单条/全部已读（数据与知识下架、获赞等写入联动） |
| Service | ContentService、ContentServiceImpl | 列表/详情/保存/更新/删除/恢复、永久删除与清空回收站、批量删除与下架/恢复发布、收藏、分享、点赞、分类 CRUD；下架时作者通知 |
| Service | UserNotificationService | 按用户分页查询通知、未读统计、已读标记 |
| Mapper | KnowledgeContentMapper、KnowledgeCategoryMapper、UserCollectionMapper、KnowledgeTagMapper、KnowledgeContentTagMapper | MyBatis-Plus BaseMapper |
| Entity | KnowledgeContent、KnowledgeCategory、UserCollection、KnowledgeTag、KnowledgeContentTag | 与表一致，KnowledgeContent 使用 @TableLogic 对应 deleted |

### 5.2 列表查询实现

- **ContentServiceImpl.list**（用户端公开列表）：条件 deleted=0、**status 固定已发布(1)**（忽略非法 status 参数）；若 categoryId 非空则 eq；若 title 非空则 like；若 tag 非空则子查询关联 `knowledge_content_tag` + `knowledge_tag` 按标签名过滤。**sortBy=hot** 时按 view_count 降序再按 create_time 降序；否则（含 latest）按 create_time 降序。返回字段含 id、title、categoryId、categoryName、cover、summary、viewCount、likeCount、createTime 等。
- **ContentServiceImpl.adminContentPage**（管理端）：deleted=0，可选 categoryId、title、**status**（空则不过滤状态），按 create_time 降序。

### 5.3 详情与状态返回

- **getDetail(id, userId)**：根据 id 查 KnowledgeContent；**若不存在或无权限查看（如非作者/非管理员访问草稿、待审核）则抛业务异常（如 404）**。组装 Map（标题、分类名、正文、封面、浏览量、点赞、作者 ID、作者名、createTime、rejectReason、tags 等）。若 userId 非空，额外返回当前用户 `liked`、`collected`；**已发布**且访问者非作者且非管理员时递增 view_count。**登录用户**访问详情成功时调用推荐模块 **VIEW 行为上报**（`recommendService.recordBehavior`，类型 CONTENT）；未登录用户仅浏览公开可见内容时不上报。

### 5.4 收藏、分享与点赞

- **collect / uncollect / collectList**：收藏去重（`user_collection` 唯一约束），支持取消收藏与收藏列表分页查询。  
- **shareUrl(contentId)**：返回 `app.client-base-url + "/#/knowledge/{id}"`，前端一键复制分享链接。  

- **toggleLike(userId, contentId)**：先查 user_content_like 是否存在；存在则取消点赞并 `knowledge_content.like_count - 1`，不存在则点赞并 `like_count + 1`。  
- 非作者本人点赞时，向内容作者写入 `user_notification` 站内通知。  
- 点赞为幂等切换，返回 `liked` 与最新 `likeCount`。

### 5.5 内容安全与编辑一致性

- **详情内容安全渲染**：前端详情页渲染前移除危险标签与事件属性，降低 XSS 风险。  
- **并发编辑保护**：编辑提交携带 `updateTime`，后端检测内容是否被他人修改，冲突时提示刷新重试。  
- **自动草稿**：**仅新增知识**时本地可恢复草稿；**编辑已发布/已有服务端内容**时不弹窗恢复本地草稿，避免覆盖已发布正文。

### 5.6 生命周期管理与审计

- **状态切换**：管理端支持发布/下架切换（1/2）；下架已发布内容且操作者非作者时，写入作者站内通知。  
- **回收站**：逻辑删除后进回收站；可恢复、可单条**彻底删除**（物理删除并清理标签关联、收藏、点赞）或**清空回收站**。  
- **批量**：管理员可勾选多条执行**批量逻辑删除**、**批量下架**（仅对已发布生效）、**批量恢复发布**（仅对已下架生效）。  
- **审计记录**：删除、恢复、下架/恢复发布、彻底删除、清空回收站等写入 `audit_log`。

### 5.7 富文本与插图

- 用户端 `client/src/views/knowledge/ContentForm.vue`、管理端 `admin/src/views/content/ContentForm.vue` 使用 `RichTextEditor`（wangEditor v5），正文为 HTML 入库。  
- 插图走 `POST /content/editor/image` 或 `POST /admin/content/editor/image`，文件保存至 `uploads/knowledge_editor/`，返回可访问 URL。  
- 用户端详情 `Detail.vue` 对正文做安全 HTML 渲染（防 XSS）。

### 5.8 管理端界面

- **知识内容**：状态筛选（全部/已发布/下架/待审核/草稿）、分类与标题查询；支持行勾选与**批量下架**、**批量恢复发布**、**批量删除**；存在待审核时默认进入待审核筛选项，否则默认已发布。  
- **回收站**：独立路由 `/content/recycle`，仅展示逻辑删除内容，支持恢复、彻底删除、清空回收站。  
- **分类管理**：独立路由 `/category`，页面 `CategoryList.vue` 提供 **查**（列表拉取 `GET /content/category/list`）、**增**（右上角「新增分类」弹窗 → `POST /admin/content/category`）、**改**（行内编辑 + `PUT /admin/content/category/{id}`）、**删**（`DELETE /admin/content/category/{id}`）；顶部为**按名称关键字查询**（前端过滤）；**点击分类名称**跳转知识内容页 `/#/content?categoryId=xxx`，列表按该分类筛选且**状态为「全部」**以查看该分类下所有知识。

### 5.9 分类管理

- **categoryList**：查询 knowledge_category 且 status=1，按 sort 升序，返回 id、name、sort。  
- **adminSaveCategory、adminUpdateCategory、adminDeleteCategory**：对 knowledge_category 的 insert、updateById、deleteById；管理端修改接口校验名称非空（若传 name）且至少更新名称或排序之一。

### 5.10 标签

- 新增/修改内容时请求体可带 `tags`（标签名列表），`ContentServiceImpl.bindTags` 维护 `knowledge_tag` 与 `knowledge_content_tag`；无单独「标签管理」后台页时，标签随内容维护。

### 5.11 站内通知与用户端「消息中心」运行说明

**概念**：「站内消息中心前端」指在**用户端（client）**为登录用户提供**集中查看站内通知**的页面与入口：列表、详情、未读提醒，与「只在业务页弹 Toast」相对。

**本项目中已实现**（不依赖单独「用户中心」大模块即可使用）：

1. **数据写入（与知识模块绑定）**  
   - 表 `user_notification`（脚本 `DB/10_user_security_and_audit.sql`）：`user_id` 为接收人、`title`、`content`、`is_read`、`create_time`。  
   - **管理员下架他人已发布内容**：`ContentServiceImpl.changeStatus` 在条件满足时 `insert` 一条通知（如标题「知识内容已下架」）。  
   - **他人点赞自己的内容**：`toggleLike` 在点赞成功且点赞者非作者时写入「内容获赞通知」。  
   - 写入失败（如表未建）时部分场景仅打日志，**主业务仍完成**（如已下架）。

2. **后端接口**（`UserNotificationController`，路径前缀 `/user/notifications`）  
   - 分页列表：支持按 `isRead` 筛未读/已读。  
   - `unread-count`：未读数量，供前端角标。  
   - 详情：校验 `user_id`，仅本人可读。  
   - `PUT .../read`、`PUT .../read-all`：已读状态更新。

3. **用户端前端运行方式**（`client`）  
   - **路由**：`#/notifications` 消息列表（`NotificationList.vue`），`#/notifications/:id` 详情（`NotificationDetail.vue`），需登录。  
   - **入口**：顶栏「消息」链到列表；个人中心 `Profile.vue` 提供「消息通知」链接。  
   - **未读角标**：`Layout.vue` 登录后定时请求 `GET /user/notifications/unread-count`（约每 8 秒），并在列表/详情内标记已读后通过自定义事件 `client-notifications-updated` 触发立即刷新角标。  
   - **列表页**：拉取分页数据，展示标题、摘要、时间；支持「全部已读」。点击进入详情后可单条已读。

**与「用户中心模块」关系**：通知的**账号归属**即 `user_id`，与个人中心展示同一登录用户；本系统将消息列表做在独立路由而非嵌在个人资料 Tab 内，功能上仍属于「登录用户站内信」能力，**已实现集中展示**，无需再单独开发一个模糊命名的「用户中心」才能完成阅读通知。

## 六、与其它模块关系

- **推荐模块**：**登录用户**打开详情成功时上报 VIEW（CONTENT、contentId）；浏览量仅在「已发布且访问者既非作者也非管理员」时递增。点赞成功时上报 LIKE。  
- **用户模块**：作者展示名通过 `author_id` 关联 `sys_user.username` 填入 authorName；**站内通知**通过 `user_notification.user_id` 与当前登录用户对应，由 **5.11** 所述接口与页面消费。

| 管理端列表：状态筛选、待审核优先、回收站独立页、行内审核与批量操作 | 已实现 |
| 并发编辑携带 updateTime、编辑已发布不恢复本地草稿 | 已实现（见 5.5） |
| 站内通知写入 + 用户端消息列表/详情/未读角标 | 已实现（见 **5.11**、`/user/notifications`） |

### 7.2 未作为必需项实现的能力（可作毕设扩展）

以下在中小型内容管理系统中常见，**本模块当前未实现**，若论文或答辩需要可列为「后续工作」：

- **独立标签管理后台**：标签随内容维护，无单独标签 CRUD 页面（见 5.10）。  
- **内容版本历史 / 操作留痕（除 audit_log 外的正文级版本）**：当前仅逻辑删除 + 审计，无多版本正文快照。  
- **全文检索接口**：表上可有 FULLTEXT，业务查询仍为标题 **LIKE**，未单独提供全文检索 API。  
- **批量导入/导出**：无 Excel/Markdown 批量导入知识、无导出接口。  
- **通知类扩展**（可选）：如按类型筛选（下架/获赞/系统）、WebSocket 实时推送、管理端代发全员公告、邮件/短信同步推送等；当前为**轮询未读数 + 列表拉取**。

以上不影响「消防知识内容管理」核心闭环：**撰写 → 提交审核 → 管理员审核 → 发布/下架 → 回收与恢复**。
