# 功能文档 - 模块二：消防知识内容管理模块

本文档对应需求与设计中的**消防知识内容管理模块**，说明内容发布与查询、分类管理、收藏与分享的**代码设计与实现**。

---

## 一、功能概述

管理员或作者可发布、更新、删除消防知识内容；支持按分类、标题模糊搜索与分页查询；用户可收藏内容、查看收藏列表并生成分享链接。分类由管理端维护（增删改）。

---

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 内容发布 | 管理员或作者更新内容保存到数据库 | POST /content 接收 title、categoryId、content、cover、tags、status；authorId 取当前登录用户 |
| 内容查询 | 按分类、标题模糊搜索、分页（HelpPage） | MyBatis-Plus 分页 + LambdaQueryWrapper 条件 categoryId、like(title)；返回 IPage 转 list/total/pageNum/pageSize |
| 内容管理 | 作者、管理员对内容增删改查 | 逻辑删除：MyBatis-Plus 配置 logic-delete，DELETE 接口调用 mapper.deleteById |
| 收藏与分享 | 用户收藏知识；新增 user_collection；生成分享链接 | 收藏：user_collection 表唯一 (user_id, content_id)；分享：返回前端配置的 client-base-url + /article/{id} |

## 四、数据库设计

脚本：`DB/02_knowledge.sql`。

| 表名 | 说明 |
|------|------|
| knowledge_category | 分类：id, name, sort, status |
| knowledge_content | 内容：id, title, category_id, content, cover, summary, author_id, view_count, like_count, status, deleted(逻辑删除), create_time, update_time；FULLTEXT(title) |
| knowledge_tag | 标签：id, name |
| knowledge_content_tag | 内容-标签多对多：content_id, tag_id |

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | ContentController | 用户端：list、detail、save、update、delete、collect、uncollect、collectList、share、categoryList |
| Controller | AdminContentCategoryController | 管理端：category 的 POST/PUT/DELETE |
| Service | ContentService、ContentServiceImpl | 列表/详情/保存/更新/删除、收藏/取消/收藏列表、分享链接、分类 CRUD |
| Mapper | KnowledgeContentMapper、KnowledgeCategoryMapper、UserCollectionMapper、KnowledgeTagMapper、KnowledgeContentTagMapper | MyBatis-Plus BaseMapper |
| Entity | KnowledgeContent、KnowledgeCategory、UserCollection、KnowledgeTag、KnowledgeContentTag | 与表一致，KnowledgeContent 使用 @TableLogic 对应 deleted |

### 5.2 列表查询实现

- **ContentServiceImpl.list**：构建 Page&lt;KnowledgeContent&gt;(pageNum, pageSize)，LambdaQueryWrapper 条件：deleted=0；若 categoryId 非空 eq(categoryId)；若 title 非空 like(title)。orderByDesc(createTime)。执行 contentMapper.selectPage，getRecords() 逐条转为 Map（含 id、title、categoryId、categoryName 通过 categoryMapper.selectById 查名）、cover、summary、viewCount、likeCount、createTime），返回 IPage&lt;Map&gt;。

### 5.3 详情与收藏状态

- **getDetail(id, userId)**：根据 id 查 KnowledgeContent；若为空返回 null。组装 Map（标题、分类名、正文、封面、浏览量、点赞、作者 ID、作者名通过 userMapper 查、createTime、tags 当前可为空列表）。若 userId 非空，查 user_collection 表是否存在 (userId, contentId)，设置 map.put("collected", true/false)。

### 5.4 收藏与去重

- **collect(userId, contentId)**：先 count 是否存在 (userId, contentId)；若为 0 则 new UserCollection 并 insert。  
- **uncollect**：按 userId、contentId 删除。  
- **collectList**：按 userId 分页查 user_collection，再根据 content_id 查知识摘要并带上 collectTime、categoryName 返回 PageResult。

### 5.5 分享链接

- **shareUrl(contentId)**：读取配置 `app.client-base-url`（如 http://localhost:5173），返回 Map：shareUrl = clientBaseUrl + "/article/" + contentId，contentId。

### 5.6 分类管理

- **categoryList**：查询 knowledge_category 且 status=1，按 sort 升序，返回 id、name、sort。  
- **adminSaveCategory、adminUpdateCategory、adminDeleteCategory**：对 knowledge_category 的 insert、updateById、deleteById。

## 六、与其它模块关系

- 内容详情可扩展：在详情接口内增加 view_count 自增、并调用推荐模块的**行为上报**（VIEW、CONTENT、contentId），用于个性化推荐。  
- 作者名：通过 author_id 查 sys_user 得到 username 填入 authorName。
