# 功能文档 - 模块六：消防新闻模块

本文档对应需求与设计中的**消防新闻模块**，说明新闻发布、列表筛选排序、详情与浏览量、**上下架（status）**、管理端分页、入参校验及**富文本 HTML 后端消毒 + 用户端安全渲染**的代码设计与实现；并包含**分类字典**、**RSS**、**评论与点赞（站内通知）**、**摘要留空自动生成**、**操作审计**、**可选 ngram 全文检索**等扩展能力，与当前仓库一致。

---

## 一、功能概述

管理员通过 **wangEditor 富文本**编辑正文（可排版、插图，图片走既有上传接口）；保存时后端对 **`content` 做 OWASP HTML 消毒**，并可按配置对 **img[src] / a[href]** 做**主机白名单**（`app.news.allowed-resource-hosts`，空则不过滤）后入库。**title** 经 **`PlainTextSanitizer`** 收敛；**summary** 若提交为空或空白，则由 **`NewsAutoSummaryUtil`** 在消毒后的正文中抽取生成再入库。**分类**优先使用 **`categoryId`**（关联 **`news_category`** 字典）；兼容历史 **`category`** 自由文本字段。用户端详情在 **DOMPurify 二次净化**后使用 **`v-html` 安全渲染**。用户端仅浏览 **status=1** 的新闻；详情访问 **view_count + 1**。**GET /news/list**、**GET /news/{id}**、**GET /news/{id}/comments** 入口侧支持按 IP 可配置限流（列表/详情阈值见 `application.yml`，**0** 关闭）。**GET /news/rss**、**POST /news/{id}/comments** 另有 RSS 与发表评论的 IP/用户限流（`news-rss-per-ip-per-minute`、`news-comment-post-per-ip-per-minute`、`news-comment-post-per-user-per-minute`，**0** 关闭）。**RSS**：**GET /news/rss** 输出最近已上架新闻的 RSS 2.0（链接与频道元数据依赖 **`app.client-base-url`**）。**评论**：登录用户可发评论/回复，触发 **`user_notification` 站内通知**（发布人或被回复人，非本人时）。**点赞**：登录用户 **POST /news/{id}/like** 幂等切换，维护 **`news_like`** 与 **`news.like_count`**。**新闻与分类字典的增删改**等管理操作写入 **`audit_log`**（与全局审计策略一致）。

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 新闻发布 | 管理员保存，publisher_id 为当前用户 | `POST/PUT`：**categoryId**（字典）+ 可选 **category**（旧文本）；**summary** 可空，空则自动生成；**coverUrl**（`NewsCoverUrlUtil`）；正文 **finalizeNewsHtml** |
| 分类字典 | 下拉维护，避免自由文本混乱 | 表 **`news_category`**；公开 **`GET /news/category/list`**；管理端 **`/admin/news/category`** CRUD；分类变更写 **audit_log** |
| 用户端列表 | 筛选与展示 | `GET /news/list`：**categoryId**、**category**（旧精确字）、**keyword**、**title**、**titleFulltext**、**searchMode**：`like`（默认）或 **`ngram`**（**MATCH** 用 **NATURAL LANGUAGE**，可与 **title**/**titleFulltext** 组合；受 **`app.news.ngram-fulltext-enabled`** 与 **(title,summary) FULLTEXT** 约束；**缺索引时自动回退**标题+摘要 **LIKE**）；响应 **categoryName**、**likeCount** 等 |
| 用户端详情 | 相关推荐与互动 | `GET /news/{id}`：**related**、**categoryName**、**likeCount**、**liked**（仅登录且带 Token 时）；评论 **GET/POST /news/{id}/comments**；点赞 **POST /news/{id}/like** |
| RSS | 站外订阅 | `GET /news/rss`，`Content-Type: application/rss+xml; charset=UTF-8` |
| 管理端列表 | 含下架稿 | `GET /admin/news/page`：与用户对齐的筛选 + **status**；可选 **categoryId**、**searchMode** |
| 管理端详情 | 编辑用 | `GET /admin/news/{id}` |
| 安全 | 存储型 XSS | **OWASP** + 用户端 **DOMPurify**；评论走 **`PlainTextSanitizer.sanitizeCommentBody`**，纯文本展示 |

## 四、数据库设计

- **`DB/06_news.sql`**：**news** 表（含 **content LONGTEXT**、**category_id**、**like_count** 等与当前代码一致字段）。  
- **`DB/13_news_category_comment_like_rss.sql`**：**news_category**、**news_comment**、**news_like**（若库由旧版 06 创建，勿在本脚本中重复 ALTER news，见 **13b**）。  
- **`DB/13b_news_alter_legacy_columns.sql`**：旧库仅为 **news** 增补 **category_id**、**like_count**。  
- **`DB/14_news_ngram_fulltext_optional.sql`**：可选 **title+summary** FULLTEXT；脚本默认**无 `ngram` 解析器**（兼容 MariaDB）。**`WITH PARSER ngram` 仅 Oracle MySQL 5.7.6+**；若建索引报 **1128**，勿使用 ngram 行，保留默认语句即可。  

执行顺序与说明见 **`DB/README.md`**。

## 五、代码设计与实现

### 5.1 依赖与工具类

- **Maven**：`owasp-java-html-sanitizer`；**`org.jsoup:jsoup`**（白名单过滤 DOM、`isEffectivelyBlank` 取文本）。  
- **`RichTextHtmlSanitizer`**：OWASP **`sanitize`**；**`isEffectivelyBlank`**：先 **`sanitize`** 再 **`Jsoup.parseBodyFragment(...).text()`** 判断是否空。  
- **`PlainTextSanitizer`**：标题/摘要/分类去 HTML；**`sanitizeCategory`**（64 字）。  
- **`NewsCoverUrlUtil`**：封面 URL 校验。  
- **`NewsMediaWhitelistFilter`**：在 OWASP 之后按 **`app.news.allowed-resource-hosts`** 处理 **img / a**（配置为空则跳过）。  
- **`NewsReadRateLimiter`**：**`NewsController`** 在 list、detail、**RSS**、**发表评论** 入口调用（含 RSS IP、评论 IP/用户；阈值 **0** 关闭）。

### 5.2 Service

- **NewsServiceImpl**：列表 **categoryId / searchMode**、**ngram** 开关与 **FULLTEXT 异常时回退 LIKE**；**titleFulltext**/**ngram** 路径下 **MATCH** 均为 **NATURAL LANGUAGE**；**getDetail** 带 **liked**；**toggleLike** 事务内 **行锁** 更新点赞与 **like_count**；**adminSave / adminUpdate** 摘要自动生成与 **audit_log**；**NewsCommentServiceImpl**、**NewsRssService**、**NewsCategoryDictServiceImpl** 各司其职。  
- **UserNotificationService**：评论/回复 **sendSimple** 站内通知（仅站内，非邮件）。

### 5.3 用户端前端

- **`sanitizeNewsHtml.js`** + **`Detail.vue`**：**v-html** 安全渲染；**点赞**、**评论列表/发表/回复**、**RSS** 链接（`getNewsRssAbsoluteUrl`）。  
- **`List.vue`**：**categoryId** 下拉、**searchMode**、**keyword**；展示 **categoryName**、**likeCount**。  
- **`client/src/api/news.js`**：列表、详情、分类、评论、点赞、RSS 地址辅助函数。

### 5.4 管理端前端

- **`NewsForm.vue`**：**categoryId** 字典下拉；摘要说明留空自动生成。  
- **`NewsList.vue`**：**categoryId**、**searchMode**、旧 **category** 字筛选；列 **categoryName**。  
- **`NewsCategoryList.vue`**：分类字典维护。  
- **`NewsCommentList.vue`**：新闻评论分页、按新闻 ID/正文关键词筛选、删除、跳转编辑新闻。  
- **`RichTextEditor`**、封面上传同前。

### 5.5 列表排序与其它

- **list / adminPage**：`applyOrder`（urgency 次级 `publishTime desc`）。  
- **parseUrgencyLevel / parsePublishTime / status** 等校验同前。

## 六、说明

- **纵深防御**：入库消毒（+ 可选外链主机白名单）+ 出库再处理 + 浏览器侧 DOMPurify；title/summary 已按纯文本处理，仍应避免对未消毒字段误用 **v-html**。  
- 若需放宽/收紧标签，应**同时**评估 **OWASP Policy** 与 **DOMPurify CONFIG** 一致性；外链治理可叠加 **CDN 白名单** 与运营规范（大体积图、钓鱼链等）。  
- 新闻列表/详情/**RSS**/**发表评论**限流为**单机内存窗口**，集群部署请改用网关或 Redis 限流。**默认**（`application.yml`）：列表 120/分钟/IP、详情 90、RSS 120、评论发表 30/IP·分钟与 20/用户·分钟（键名见 **`app.rate-limit.news-*`**）；**`dev`** 下 `application-dev.yml` 将上述及 RSS、评论项均置 **0** 便于本地调试。  
- 管理端列表须使用 **`/admin/news/page`** 管理下架稿。  
- **FULLTEXT** 对中文分词：无 ngram 解析器时与默认英文分词类似，中文场景仍建议优先 **keyword** + **searchMode=like**。执行 **`14`** 并 **`app.news.ngram-fulltext-enabled: true`** 后可用 **searchMode=ngram**（**NATURAL LANGUAGE** 下 **`MATCH(title,summary)`**）；**MariaDB** 用脚本默认句，勿 **`WITH PARSER ngram`**（**1128**）。**Oracle MySQL 5.7.6+** 可选脚本内 ngram 行改善中文短语。**titleFulltext** 对应 **`MATCH(title)`**（同为 NATURAL LANGUAGE）。  
- **Elasticsearch** 等专业全文检索未在本仓库强制引入；若后续接入，可在网关或检索服务层替换列表查询实现，接口参数可沿用 **keyword/searchMode** 语义扩展。
