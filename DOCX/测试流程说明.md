# 消防科普推荐系统 — 整体测试流程说明

本文档描述从环境准备到各模块的**推荐测试顺序**与**要点**，与《后端接口文档》**§1.5**（安全/限流）、各《功能文档》及 [功能文档索引.md](功能文档索引.md) 对齐。

---

## 一、环境准备

1. **数据库**：执行 `DB/` 下必要脚本（至少用户/角色、知识、论坛、器材、新闻等），保证存在 **ADMIN** 账号或可注册后由管理员创建。  
2. **Redis**（可选）：论坛点赞持久化依赖 Redis；未启动 Redis 时论坛点赞不按用户切换，属预期。  
3. **后端**：`backend` 启动，默认 `http://localhost:8088/api`；检查 `application.yml` 中数据源、Redis、邮件（找回密码联调用）。  
4. **前端**：  
   - 用户端 `client`：`npm install` → `npm run dev`（常见端口 5177）。  
   - 管理端 `admin`：`npm install` → `npm run dev`（常见端口 5173）。  
5. **环境变量 / 配置**：管理端、用户端 `.env` 或 `VITE_API_BASE_URL` 指向上述 API 根路径。

## 二、推荐测试顺序（端到端）

### 阶段 A：账号与权限

1. **用户注册** `POST /user/register`（或用户端注册页）→ 登录 `POST /user/login`，保存 token。  
2. **限流抽检**（可选）：短时间重复注册/登录，应出现「请求过于频繁」类提示（见 §1.5）。  
3. **管理端登录** `POST /admin/login`（仅 ADMIN）→ 非管理员应 403。  
4. **末位管理员**（需至少两个管理员或先建测试账号）：尝试删除/冻结最后一个可登录管理员 → 应业务失败；尝试删除当前登录的自己 → 应失败。

### 阶段 B：知识内容（模块二）

1. 普通用户 **申请作者** → 管理员 **通过作者申请**。  
2. 作者 **新增知识草稿** → **提交审核** → 管理员 **审核通过**。  
3. 用户端 **公开列表**仅见已发布；**详情**访问已发布内容，浏览量变化符合预期。  
4. 管理员 **下架**该内容 → **未登录/非作者**访问详情应 **404**；作者或管理员仍可访问。  
5. **回收站 / 恢复 / 彻底删除**（管理端）抽测一条闭环。

### 阶段 C：论坛（模块三）

1. 用户 **发帖** → 状态待审核；**用户端帖子列表**不应出现该帖。  
2. **管理端帖子审核列表** `GET /admin/forum/post/list?status=0` 可见；**通过**后用户端列表出现。  
3. **未通过帖**：匿名访问详情/评论列表 → **404**；**作者带 Token** 可访问。  
4. **已通过帖**：点赞、评论成功；对未通过帖点赞/评论 → 业务错误。  
5. **推荐行为表**（可选查库）：论坛点赞、评论是否产生 **FORUM_POST** 类型行为记录。

### 阶段 D：推荐（模块四）

1. **未登录**调用 `GET /recommend/list`：应为**冷启动**（无个人画像相关因子；响应 **pageHint** 可说明策略）。  
2. **登录**后再次调用：有个人行为时应体现召回 + 多因子排序（可与「按时间」「热门」Tab 列表对比）。  
3. **行为上报** `POST /recommend/behavior`：合法枚举成功；非法 `behaviorType`/`targetType` 或 `targetId≤0` → 失败；高频调用触发限流。  
4. **用户端**：「消防知识 → 智能推荐」**课堂演示模式**（记录 Top5 → 注入行为 → 刷新对比）；**个人中心 → 推荐偏好** 设置不感兴趣分类后，智能推荐列表应过滤该分类。  
5. **AI**：在服务端 `ai.recommend.*` 就绪前提下，请求 **`useAi=true`**（用户端勾选「AI 增强」）且**已登录**，观察 `recommendReason` 与头部顺序变化；`useAi=false` 或不传应始终为规则序；失败或超时**静默降级**。  
6. （可选）`POST /recommend/feedback` 与表 `recommend_feedback`：需执行 `DB/17_recommend_feedback_optional.sql`。

### 阶段 E：器材（模块五）

1. **类型管理**：管理端「器材类型」增删改；类型被器材引用时删除应失败。  
2. **器材 CRUD**；修改**不存在**的器材 ID → **404**。  
3. **Excel 导入**：按文档表头准备 xlsx → 导入成功/失败统计与 errors 提示。

### 阶段 F：新闻（模块六）

1. 管理端发布新闻；用户端列表筛选与 **orderBy + order** 切换。  
2. 详情与管理员编辑、删除抽测。

### 阶段 G：统计（模块七）

1. 管理员进入 **数据统计** 页：**全站运营大盘**卡片、内容/用户/互动卡片、**饼图**是否正常。  
2. **日期区间**：选择起止后刷新，饼图与大盘随区间变化；**折线图**为区间内按月双序列（知识 view 汇总 vs 行为 VIEW），清空区间时为**当年**。  
3. **导出 Excel**：**导出全部**时 xlsx 除内容/用户/互动外应含**运营大盘**与**月度趋势**行；网络面板成功时为文件流；**仅传单日**参数应返回业务错误（须成对日期）。  

### 阶段 H：安全与部署（可选）

1. **prod 配置**：`spring.profiles.active=prod` 时文档接口 **404**、`app.api-docs.enabled=false`。  
2. **CORS**：生产 `app.cors.allow-all=false` 时，仅允许来源的前端可跨域带凭证访问。  
3. **找回密码**：`sendEmail` 邮箱限频 + IP 限频是否生效。

## 三、接口自动化 / 工具建议

- 使用 **Knife4j**（开发环境 `http://localhost:8088/api/doc.html`）单接口调试。  
- 或使用 **Postman / Apifox**：导入 OpenAPI（若开启）。  
- **导出与文件类接口**必须用支持 **二进制** 的模式验证。
