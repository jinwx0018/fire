# 消防科普推荐系统 — 前端接口文档

本文档根据《毕业设计需求文档》及各功能模块文档整理，供**用户端（client）**与**管理端（admin）**前端联调使用。

---

## 一、通用说明

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 系统名称 | 基于 SpringBoot 的消防科普推荐系统 |
| 后端技术 | Spring Boot、JWT、MyBatis-Plus |
| 前端项目 | **client**（用户端）、**admin**（管理端） |

### 1.2 请求基础地址

开发环境示例：`http://localhost:8088/api`（与《文件夹说明》及 `application` 配置一致；若端口不同以实际为准）

### 1.3 认证方式

- **需认证接口**：请求头携带 `Authorization: Bearer <accessToken>`
- **会话治理**：登录后同时保存 `token` 与 `refreshToken`，401 时调用 `/user/refresh-token` 自动刷新并重试请求；刷新失败则清理登录态并跳转登录页
- **无需认证**：注册、登录、部分公开查询接口（如知识列表、新闻列表、器材列表等）

### 1.4 统一响应格式（建议）

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | 业务状态码，如 200 成功、400 参数错误、401 未登录、403 无权限 |
| message | string | 提示信息 |
| data | object/array | 业务数据；分页时可能为 `{ list, total, pageNum, pageSize }` 或 MyBatis-Plus 的 **`records, total, current, size, pages`** |

分页列表统一结构示例（两种形态择一，以接口实际返回为准）：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 0,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 0,
    "current": 1,
    "size": 10,
    "pages": 0
  }
}
```

### 1.5 分页参数（列表类接口通用）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 10 |

### 1.6 限流、文档与文件下载（与后端一致）

- **注册 / 登录（含 `/admin/login`）/ 发找回密码邮件**：后端按 IP、用户名等维度限频，频繁请求返回 `code≠200` 且 `message` 含「过于频繁」类提示。联调时勿用脚本高并发压测匿名接口。
- **行为上报** `POST /recommend/behavior`：`behaviorType`、`targetType` 须为后端允许枚举；每分钟每用户有上限。
- **统计导出** `GET /admin/statistics/export`：响应为 **Blob**，axios 需 `responseType: 'blob'`（管理端 `request` 已对 blob 做特殊处理）。
- **管理端帖子审核列表**：请求 **`GET /admin/forum/post/list`**（勿再对 `/forum/post/list` 传 `status` 筛审核数据）。
- 生产环境关闭在线文档时，访问 `/doc.html` 可能 **404**，属预期。

## 二、模块一：用户管理

### 2.1 用户注册

| 项目 | 说明 |
|------|------|
| **方法** | POST |
| **路径** | `/user/register` |
| **认证** | 否 |

**请求体 (application/json)**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，唯一 |
| password | string | 是 | 密码，前端可做强度校验 |
| phone | string | 是 | 手机号，唯一 |
| email | string | 否 | 邮箱 |

**响应示例**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": 1
}
```
`data` 为新增用户 ID。失败时 `message` 为「用户名已存在」「手机号已注册」等。

## 八、模块七：数据统计与分析（管理端）

以下接口均需管理员认证。

**日期约定（与后端一致）**：除 **viewTrendRange** 在未选日期时按**当年**外，凡传 **startDate/endDate** 的接口均须**成对传入**，且区间不超过 **3650** 天；否则接口返回业务错误。详见《后端接口文档》第八章开篇。

**管理端页面（admin）**：`Statistics.vue` 日期选择器**清空**时不传日期；**选定区间**时始终传**起止两天**，与上述约定一致；**互动**指标为**仅已通过**论坛帖及下属评论、点赞合计（见 **8.3**）。**导出全部**时后端附带运营大盘与月度趋势行（**type** 空或 **all**）。

## 十、错误码与异常（建议约定）

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误、业务校验失败（如用户名已存在） |
| 401 | 未登录或 Token 无效/过期 |
| 403 | 无权限（如非管理员访问管理接口） |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

前端可根据 `code` 统一处理：401 跳转登录、403 提示无权限、4xx/5xx 展示 `message`。
