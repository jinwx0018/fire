# 消防科普推荐系统 — 前端接口文档

本文档根据《毕业设计需求文档》及各功能模块文档整理，供**用户端（client）**与**管理端（admin）**前端联调使用。

---

## 一、通用说明

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 系统名称 | 基于 SpringBoot 的消防科普推荐系统 |
| 后端技术 | Spring Boot、JWT、MyBatis-Plus |
| 前端项目 | **client**（用户端）、**admin**（管理端） |

### 1.2 请求基础地址

开发环境示例：`http://localhost:8080/api`（以实际后端配置为准）

### 1.3 认证方式

- **需认证接口**：请求头携带  
  `Authorization: Bearer <JWT Token>`
- **无需认证**：注册、登录、部分公开查询接口（如知识列表、新闻列表、器材列表等）

### 1.4 统一响应格式（建议）

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | 业务状态码，如 200 成功、400 参数错误、401 未登录、403 无权限 |
| message | string | 提示信息 |
| data | object/array | 业务数据，分页时为 `{ list, total, pageNum, pageSize }` |

分页列表统一结构示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 0,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

### 1.5 分页参数（列表类接口通用）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 10 |

## 二、模块一：用户管理

### 2.1 用户注册

| 项目 | 说明 |
|------|------|
| **方法** | POST |
| **路径** | `/user/register` |
| **认证** | 否 |

**请求体 (application/json)**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，唯一 |
| password | string | 是 | 密码，前端可做强度校验 |
| phone | string | 是 | 手机号，唯一 |
| email | string | 否 | 邮箱 |

**响应示例**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": 1
}
```
`data` 为新增用户 ID。失败时 `message` 为「用户名已存在」「手机号已注册」等。

---

### 2.2 用户登录

| 项目 | 说明 |
|------|------|
| **方法** | POST |
| **路径** | `/user/login` |
| **认证** | 否 |

**请求体 (application/json)**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**响应示例**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "userId": 1,
    "username": "zhangsan",
    "role": "USER",
    "avatar": "https://xxx/avatar.png"
  }
}
```

前端需将 `token` 存入本地（如 localStorage），后续请求头携带 `Authorization: Bearer <token>`。

## 十、错误码与异常（建议约定）

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误、业务校验失败（如用户名已存在） |
| 401 | 未登录或 Token 无效/过期 |
| 403 | 无权限（如非管理员访问管理接口） |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

前端可根据 `code` 统一处理：401 跳转登录、403 提示无权限、4xx/5xx 展示 `message`。
