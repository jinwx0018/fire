# 功能文档 - 模块五：消防器材查询模块

本文档对应需求与设计中的**消防器材查询模块**，说明器材录入、按名称与类型查询分页、详情展示、**上下架（status）**、**批量导入**、**封面/多图 URL 安全校验**及**管理端分页接口**的代码设计与实现，与当前仓库实现保持一致。

---

## 一、功能概述

管理员可新增、修改、删除器材及器材类型，支持 Excel 批量导入；器材支持 **上架（status=1）/ 下架（status=0）**，用户端**仅展示已上架**记录。用户端支持按名称模糊搜索、按类型筛选、分页列表与图文详情（封面、多图、使用步骤、检查要点、常见故障解决）。

**安全与运维**：封面与多图 URL 在保存/导入时校验（**仅 http(s) 公网地址**，或**本站相对路径** `/xxx`，禁止 `..`、内网与 `localhost` 等）；Excel 单次导入**不超过 2000 行**；上传大小受 **`spring.servlet.multipart`** 限制，超限由全局异常处理返回明确提示。

---

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 器材录入 | 管理员录入；支持批量导入 | 单条：`POST /admin/equipment`；批量：`POST /admin/equipment/import`（EasyExcel） |
| 上下架 | 用户端仅见上架 | `equipment.status`：**1 上架**、**0 下架**；用户端 `list`/`getDetail` 仅 **status=1**；管理端分页可查全部 |
| 器材查询（用户端） | 名称模糊、类型筛选、分页 | `GET /equipment/list`：`like(name)`、`eq(typeId)`、`eq(status,1)`、分页、**typeName** |
| 器材查询（管理端） | 含下架、可筛状态 | `GET /admin/equipment/page`：`keyword`、`typeId`、可选 **`status`**（空=全部） |
| 详情（用户端） | 图文详情 | `GET /equipment/{id}`：**仅已上架**；否则 **NOT_FOUND** |
| 详情（管理端） | 编辑表单 | `GET /admin/equipment/{id}`：未删除即可，返回含 **status** |
| URL 安全 | 降低恶意链接与内网 SSRF 风险 | `EquipmentMediaUrlValidator`：校验 **cover**、**images**（JSON 数组或多 URL 分隔）；导入时校验 **封面URL** |
| 上传限制 | Excel 体积 | `max-file-size` / `max-request-size`；`MaxUploadSizeExceededException` 友好提示 |

## 三、接口列表

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | /equipment/list | 分页；keyword、typeId；**仅 status=1** | 否 |
| GET | /equipment/{id} | 详情；**仅 status=1**，否则 NOT_FOUND | 否 |
| GET | /equipment/type/list | 类型列表（id、name、sort） | 否 |
| GET | /admin/equipment/page | 管理端分页；keyword、typeId、**status 可选** | 管理员 |
| GET | /admin/equipment/{id} | 管理端详情（含下架） | 管理员 |
| POST | /admin/equipment | 新增；body 含 **status**（0/1，默认 1）、**images** 等 | 管理员 |
| PUT | /admin/equipment/{id} | 修改；可更新 **status**、**images** | 管理员 |
| DELETE | /admin/equipment/{id} | 删除（软删） | 管理员 |
| POST | /admin/equipment/import | 批量导入；**≤2000 行**；封面 URL 参与校验 | 管理员 |
| POST | /admin/equipment/type | 新增类型 | 管理员 |
| PUT | /admin/equipment/type/{id} | 修改类型 | 管理员 |
| DELETE | /admin/equipment/type/{id} | 删除类型（有器材引用则失败） | 管理员 |

## 四、数据库设计

脚本：`DB/05_equipment.sql`。

| 表名 | 说明 |
|------|------|
| equipment_type | id, name, sort, create_time, update_time |
| equipment | id, name, type_id, cover, usage_steps, check_points, fault_solution, images, summary, sort, **status（0 下架 / 1 上架）**, deleted, … |

可选：`FULLTEXT(name)` 仅作检索优化；**当前列表查询使用 `LIKE`**，与全文索引无冲突。

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | EquipmentController | 用户端：list、detail、typeList |
| Controller | AdminEquipmentController | **/page**、**/{id} 详情**、save、update、delete、import、类型 CRUD |
| Service | EquipmentService、EquipmentServiceImpl | **adminPage**、**adminGetDetail**、**getDetail**（上架校验）、导入与校验 |
| Util | EquipmentMediaUrlValidator | 封面/多图 URL 与相对路径规则 |
| Entity | Equipment、EquipmentType | `@TableLogic` |
| Exception | GlobalExceptionHandler | **MaxUploadSizeExceededException** 提示 |

### 5.2 列表与详情

- **list（用户端）**：`deleted=0` 且 **`status=1`**；keyword、typeId；`toUserListRow` 含 typeName、summary、cover。  
- **adminPage（管理端）**：`deleted=0`；**status 可选精确匹配**；`toAdminListRow` 额外 **status**。  
- **getDetail（用户端）**：存在且 **`status=1`**，否则 **NOT_FOUND**；`equipmentToMap`。  
- **adminGetDetail**：未删除即可；Map 含 **status**。

### 5.3 管理端增删改

- **adminSave / adminUpdate**：**name、typeId** 校验；**status** 仅允许 0/1；写入前 **`EquipmentMediaUrlValidator.validateCoverAndImages`**。  
- **adminDelete**：逻辑删除。  
- **类型 CRUD**：**sort** 非法数字抛业务异常；删除类型前检查器材引用。

### 5.4 批量导入（EasyExcel）

- **DTO**：`EquipmentImportRow`，表头：`名称`、`类型`、`摘要`、`使用步骤`、`检查要点`、`故障解决`、`封面URL`。  
- 行数 **>2000** 拒绝；每行 **封面URL** 走 **`validateOptionalUrl`**。  
- 导入记录默认 **status=1**（若需下架请管理端编辑）。

### 5.5 配置

- **`application.yml`**：`spring.servlet.multipart.max-file-size`、`max-request-size`（兼顾头像与 Excel）。  
- 超限：`GlobalExceptionHandler` 返回 **400** 与说明文案。

### 5.6 管理端前端

- **器材列表**：`EquipmentList.vue` → **`getEquipmentAdminPage`**（`/admin/equipment/page`）；keyword、typeId、**上架筛选**、分页、Excel 导入、状态列。  
- **器材表单**：`EquipmentForm.vue` → **上架状态**、**多图 URL**；详情 **`GET /admin/equipment/{id}`**。  
- **类型**：`EquipmentTypeList.vue`，路由 `/equipment/types`。

### 5.7 用户端前端

- **列表/详情**：`client/src/views/equipment/List.vue`、`Detail.vue`；仍使用 **`/equipment/list`**、**`/equipment/{id}`**（仅上架）。

---

## 六、注意事项

- 管理端列表**勿再调用**用户端 `/equipment/list` 作为「全部器材」数据源，应使用 **`/admin/equipment/page`**。  
- 多图字段可为 **JSON 字符串数组** 或 **逗号/换行/分号** 分隔的多个 URL。  
- 相对路径仅允许以 **`/`** 开头且不含 **`..`**，便于 `/uploads/` 等资源。

---

## 七、前端展示建议

详情页以分段或 `pre` 展示步骤/要点/故障；封面与多图列表展示；富文本若后续引入需做 XSS 过滤。
