# 消防科普推荐系统 — 后端接口文档

## 一、通用说明

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 基础路径 | `http://{host}:{port}/api`（示例：`http://localhost:8088/api`） |
| 数据格式 | 请求/响应均为 JSON（`Content-Type: application/json`） |
| 认证方式 | 除登录/注册/密码找回等接口外，需在请求头携带：`Authorization: Bearer {token}` |

### 1.2 统一响应结构

**成功：**

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

**失败：**

```json
{
  "code": 400,
  "message": "错误描述",
  "data": null
}
```

**分页响应：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10
  }
}
```

### 1.3 通用分页请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |

### 1.4 分页响应字段说明（MyBatis-Plus）

部分接口直接返回 `IPage<T>`，JSON 序列化字段为 **`records`**（列表数据）、**`total`**、**`current`**（当前页）、**`size`**（每页条数）、**`pages`**（总页数）。与下文示例中的 `list` / `pageNum` 含义对应，联调时以实际响应为准。

### 1.5 安全、限流与部署相关说明（新增）

| 类别 | 说明 |
|------|------|
| **匿名接口限流** | `POST /user/register`：按客户端 IP **每小时**限制次数（默认 30，配置 `app.rate-limit.register-per-ip-per-hour`）。`POST /user/login`、`POST /admin/login`：按 IP 与用户名 **每分钟**限制（`login-per-ip-per-minute`、`login-per-username-per-minute`；管理端登录 IP 键独立）。超限返回业务错误，提示「请求过于频繁」。 |
| **找回密码** | 除原有「同邮箱 60 秒内、每小时 5 次」外，增加按 IP **每小时**限制（`reset-email-per-ip-per-hour`，默认 20）。 |
| **行为上报** | `POST /recommend/behavior`：`behaviorType` 仅允许 `VIEW`/`LIKE`/`COMMENT`/`COLLECT`；`targetType` 允许 `CONTENT`/`FORUM_POST`/`POST`（`POST` 会规范为 `FORUM_POST`）；`targetId` 须为正数。每分钟每用户默认可调用 120 次（`app.rate-limit.behavior-per-minute`）。 |
| **接口文档访问** | `app.api-docs.enabled=false` 时，`/doc.html`、`/v3/api-docs`、`/webjars/**` 由拦截器直接 **404**。`app.api-docs.restrict-to-internal-network=true` 时仅本机或常见私网 IP 可访问文档，否则 **403**（反代需可信 `X-Forwarded-For`）。生产建议 `spring.profiles.active=prod` 并关闭 springdoc/knife4j（见 `application-prod.yml`）。 |
| **JWT 匿名路径扩展** | 可通过 `jwt.extra-exclude-paths`（逗号分隔）追加无需 Token 的路径（不含 context-path `/api` 前缀）。 |
| **CORS** | `app.cors.allow-all`：开发默认可为 `true`；生产建议 `false` 并配置 `app.cors.allowed-origin-patterns`（逗号分隔域名）。 |

**文件流响应**：`GET /admin/statistics/export` 返回 **Excel 二进制流**（非 JSON 包装）；前端需 `responseType: 'blob'` 下载。

## 二、模块一：用户管理

### 2.1 用户注册

| 项目 | 说明 |
|------|------|
| 路径 | `POST /user/register` |
| 说明 | 校验用户名、手机号格式及唯一性，密码加密入库；**按 IP 限流**（见 1.5） |

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 明文密码 |
| phone | string | 是 | 手机号 |
| email | string | 否 | 邮箱 |

**响应 data：**

```json
{
  "userId": 1,
  "username": "zhangsan"
}
```

---

### 2.2 用户登录

| 项目 | 说明 |
|------|------|
| 路径 | `POST /user/login` |
| 说明 | 用户名+密码校验，返回 accessToken + refreshToken；**按 IP 与用户名限流**（见 1.5） |

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**响应 data：**

```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
  "userId": 1,
  "username": "zhangsan",
  "role": "USER",
  "avatar": "http://xxx/avatar.jpg"
```

## 五、模块四：个性化知识推荐

本模块对**已上架知识**采用「多路召回 → 多因子精排 → 分页」：**已登录**用户综合分类偏好、Item-CF、热度/时效、可选语义与论坛跨域等信号；**未登录**为冷启动（无个人画像、无 Item-CF/语义/论坛加成）。**AI 增强**为可选：须请求参数 **`useAi=true`** 且用户已登录、服务端 `ai.recommend.*` 就绪，才对规则序**前 N 条**重排并为**当前页**生成 **recommendReason**；否则规则排序，失败或超时静默降级。用户端默认不传 `useAi`。详见《功能文档-04-个性化知识推荐模块》。

### 5.1 行为上报（可选由前端显式调用或由浏览/点赞/收藏等接口内部调用）

| 项目 | 说明 |
|------|------|
| 路径 | `POST /recommend/behavior` |
| 认证 | 需要 Token |
| 说明 | 记录浏览、收藏、点赞、评论等行为，用于兴趣分析 |

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| behaviorType | string | 是 | 仅允许 **VIEW / LIKE / COMMENT / COLLECT**（大小写不敏感，服务端规范为大写） |
| targetType | string | 是 | **CONTENT / FORUM_POST / POST**（`POST` 与 `FORUM_POST` 等价，入库为 FORUM_POST） |
| targetId | long | 是 | 目标主键，须 **> 0** |

**响应 data：** 无或 `null`

**说明：** 限流与枚举校验见 **1.5**；论坛点赞/评论亦可由业务接口内部写入行为，无需重复上报。

### 5.2 推荐列表（分页）

| 项目 | 说明 |
|------|------|

| 认证 | **JWT 可选**（`JwtInterceptor` 对 `/recommend/list` 为可选认证：无 Token 时 `userId` 为空，走冷启动） |

## 八、模块七：数据统计与分析

**日期参数统一约定（适用于除 8.5、8.6 默认当年以外的带日期筛选接口：8.1～8.4、8.7、8.8）：**

- **startDate** 与 **endDate** 须**同时传入或同时不传**。仅传其一返回 **400** 业务错误（提示须成对传入）。
- 均传时：按知识等业务统一的 **create_time** **左闭右开**区间 **[start 当日 00:00, end 次日 00:00)**（即含结束日全天）。
- 区间跨度（含首尾日）**不得超过 3650 天**（约 10 年），超出返回 **400**。
- **8.6 viewTrendRange**：均未传 **startDate/endDate** 时不走上述成对校验，后端按**当前自然年**作为默认区间。
