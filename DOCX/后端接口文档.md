# 消防科普推荐系统 — 后端接口文档

## 一、通用说明

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 基础路径 | `http://{host}:{port}/api`（示例：`http://localhost:8088/api`） |
| 数据格式 | 请求/响应均为 JSON（`Content-Type: application/json`） |
| 认证方式 | 除登录/注册/密码找回等接口外，需在请求头携带：`Authorization: Bearer {token}` |

### 1.2 统一响应结构

**成功：**

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

**失败：**

```json
{
  "code": 400,
  "message": "错误描述",
  "data": null
}
```

**分页响应：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10
  }
}
```

### 1.3 通用分页请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |

## 二、模块一：用户管理

### 2.1 用户注册

| 项目 | 说明 |
|------|------|
| 路径 | `POST /user/register` |
| 说明 | 校验用户名、手机号格式及唯一性，密码加密入库 |

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 明文密码 |
| phone | string | 是 | 手机号 |
| email | string | 否 | 邮箱 |

**响应 data：**

```json
{
  "userId": 1,
  "username": "zhangsan"
}
```

---

### 2.2 用户登录

| 项目 | 说明 |
|------|------|
| 路径 | `POST /user/login` |
| 说明 | 用户名+密码校验，返回 JWT Token |

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**响应 data：**

```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "userId": 1,
  "username": "zhangsan",
  "role": "USER",
  "avatar": "http://xxx/avatar.jpg"
}
```

---

### 2.3 获取当前用户信息

| 项目 | 说明 |
|------|------|
| 路径 | `GET /user/info` |
| 认证 | 需要 Token |

**响应 data：**

```json
{
  "userId": 1,
  "username": "zhangsan",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "avatar": "http://xxx/avatar.jpg",
```

| 项目 | 说明 |

## 五、模块四：个性化知识推荐

本模块在规则推荐（有行为/无行为策略）基础上，**支持可选接入 AI 模型**：配置 `ai.recommend.enabled`、`api-url`、`api-key` 后，推荐列表返回前可调用外部 AI/推荐服务为每条结果补充推荐理由（recommendReason）或按模型返回顺序重排序；未配置或调用失败时自动降级为规则推荐，不影响接口可用性。详见《功能文档-04-个性化知识推荐模块》。

### 5.1 行为上报（可选由前端显式调用或由浏览/点赞/收藏等接口内部调用）

| 项目 | 说明 |
|------|------|
| 路径 | `POST /recommend/behavior` |
| 认证 | 需要 Token |
| 说明 | 记录浏览、收藏、点赞、评论等行为，用于兴趣分析 |

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| behaviorType | string | 是 | VIEW / LIKE / COMMENT / COLLECT |
| targetType | string | 是 | CONTENT / POST |
| targetId | long | 是 | 知识 ID 或帖子 ID |

**响应 data：** 无或 `null`

---

### 5.2 推荐列表（分页）

| 项目 | 说明 |
|------|------|
| 路径 | `GET /recommend/list` |
| 认证 | 需要 Token（未登录可返回全站热门） |
| 说明 | 有行为则推高兴趣分类下的高浏览量；无行为则全站热门（浏览量+点赞综合）。**可选接入 AI 模型**：启用后列表项可能含 `recommendReason`（推荐理由）或按 AI 返回顺序重排。 |

**请求参数：** pageNum、pageSize

**响应 data：** 分页结构，list 为知识条目（同 3.1 列表项；启用 AI 增强时每项可能多出 **recommendReason** 字符串）。

---

## 九、状态码约定（建议）

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误/业务校验失败 |
| 401 | 未登录或 Token 无效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

以上接口与《毕业设计需求文档》中七大模块一一对应，可直接用于前后端联调与实现。
