**模块一：用户管理模块**

用户注册：前端校验普通用户名，手机号格式查重，后端再次校验唯一性，使用 BCrypt 等加密算法对密码进行加密存入数据库。

用户登录：用户名、密码校验，生成 accessToken 与 refreshToken 返回前端；管理端使用 /admin/login 且仅 ADMIN 可登录。

信息管理：用户能够修改头像、手机号、邮箱，将参数传给后端进行校验，使用 SQL 语句/ORM 更新数据库。

密码找回：支持邮箱链接找回（使用 JavaMail/Spring Mail 发送重置链接），链接 30 分钟有效，发送频率受限（60 秒内不可重复、1 小时最多 5 次），Token 一次性使用。

作者权限申请与审核：普通用户在用户端提交“申请成为作者”（支持申请原因与附件），管理端查看作者申请列表并审核（支持审核备注与驳回原因）；审核通过后将用户角色由 USER 调整为 AUTHOR。审核结果通过站内通知/邮件下发。

账号与会话治理：支持 refresh token 刷新、主动退出所有设备、管理员强制下线；修改密码、冻结、注销后旧 token 失效。账号注销采用软删除（deleted 标记）。

审计与追踪：记录注册、登录、改密、找回密码、角色变更、审核操作等关键审计日志，便于追责和问题排查；管理员可在后台分页查询审计记录（按操作类型、操作人筛选）。

站内通知与用户自助注销：用户可查看站内消息列表（未读数、已读标记），并进入**消息详情页**阅读完整正文；支持在用户端自助注销账号（密码校验、软删除），管理员账号不可通过用户端自助注销；作者申请可上传附件（图片/PDF），URL 写入申请参数。

**模块二：消防知识内容管理模块**

内容发布与作者审核流程：管理员或作者能够更新内容并保存到数据库。作者发布时先保存为草稿（status=0），点击“提交审核”后状态变为待审核（status=3），仅管理员审核通过后才会变为已发布（status=1）在用户端展示；若管理员驳回，则状态回到草稿并记录驳回原因（reject_reason），作者在“我的内容”中可以看到最近一次驳回原因并据此修改重提。

内容查询：支持按分类、标题进行模糊搜索分页查询（HelpPage）。

内容管理：作者、管理员可操作对内容进行增删改查；管理员可通过后台接口对待审核内容进行通过/驳回审核。

内容收藏与分享：用户可收藏知识内容（新增 user_collection 收藏表），支持生成分享链接。

**模块三：消防知识论坛模块**

帖子与状态：用户发布帖子（标题、正文），新建记录 **status=0（待审核）**；**用户端列表接口固定仅返回已通过（status=1）**，不接受任意 status 参数。状态含义：**1 已通过**、**0 待审核**、**-1 已驳回**；管理员审核 **通过/驳回** 时更新 status 与 reject_reason，**驳回须填写驳回理由**。作者 **修改帖子** 后系统 **将 status 置回 0** 并清空驳回理由，需再次审核通过后才会出现在公开列表。

可见性与安全：**已通过**帖子的详情与评论任何人可访问（详情访问时 **view_count 自增**）；**待审核/已驳回**帖子的详情与评论仅 **作者本人** 或 **管理员（JWT）** 可访问，否则返回 **404**。**GET /forum/comment/list** 经 JWT 拦截器按「可选认证」处理，匿名可看已通过帖的评论，作者/管理员带 Token 方可查看自己未通过帖的评论。

评论与计数：支持一级评论及 **parent_id** 回复（后端校验父评论存在且属于同一帖子）；仅 **已通过** 帖子可点赞、评论；发表评论后 **comment_count +1**，用户或管理员 **软删评论** 后 **comment_count 相应减 1**（不小于 0）。点赞采用 **Redis Set**（key：`forum:post:like:{postId}`）记录点赞用户并与 **like_count** 同步；**点赞成功**与 **发表评论** 均写入推荐行为（**LIKE/COMMENT + FORUM_POST**）。未配置 Redis 时点赞持久化与切换行为以实现为准，生产环境建议启用 Redis。

管理端与用户端操作：管理端可按 status、keyword 分页查询帖子并审核；提供 **软删帖子**、**软删评论** 接口。用户端支持本人 **删帖、删评**；接口层对 postId、parentId、审核 status 等做校验，避免非法参数导致异常。

**模块四：个性化知识推荐模块**

行为采集：记录用户浏览知识、点赞、评论帖子、收藏内容等行为，更新到用户行为表中；

兴趣分析：统计用户最常访问的分类（Top1）、互动最多的知识标签，查询用户行为表；

推荐计算：优先推荐用户高兴趣分类下的高浏览量；无行为数据时，推荐全站热门知识（浏览量 + 点赞数综合排序）；结果分页返回推荐列表。

智能推荐 - AI 模型接入：在规则推荐基础上，可选接入外部 AI/推荐模型服务（配置 ai.recommend.enabled、api-url、api-key）；调用 AI 接口为每条推荐结果补充推荐理由（recommendReason）或按模型返回顺序重排序，提升可解释性与精准度；未配置或调用失败时自动降级为规则推荐，不影响可用性。

**模块五：消防器材查询**

器材与类型维护：管理员维护器材类型（增删改），单条录入或 **Excel 批量导入**（表头固定、单次不超过 2000 行）；器材支持 **上架/下架（status）**，用户端列表与详情**仅展示已上架**；管理端使用独立分页接口查看**含下架**的全部器材并可按状态筛选。

查询与展示：用户端支持名称模糊、类型筛选与分页；详情包含封面、多图、摘要、使用步骤、检查要点与故障解决。封面与多图 URL 保存时校验（**公网 http(s)** 或本站**相对路径**，拦截内网与危险 scheme）；上传体积受 **Spring multipart** 限制，超限返回明确提示。

详细接口与安全规则见 `DOCX/功能文档-05-消防器材查询模块.md`。

**模块六：消防新闻模块**

新闻发布与状态：管理员维护标题、正文、摘要、地区、**紧急等级（1–3）**、发布时间、**上架/下架**；`publisher_id` 取当前登录用户；新建时发布时间缺省为当前时间。

用户端仅展示 **已上架** 新闻：列表支持地区与标题筛选、按发布时间或紧急等级排序、分页；详情**仅已上架**可访问，并**递增浏览量**。管理端通过 **`/admin/news/page`** 分页查看**含下架**稿件并筛选状态，编辑走 **`/admin/news/{id}`** 详情接口。

安全与校验：保存/更新时对标题、正文、等级、状态、发布时间做校验；正文使用 **OWASP Java HTML Sanitizer** 入库，接口返回前再次消毒；用户端用 **DOMPurify** 后 **v-html** 渲染，并与管理端 **wangEditor** 配合，形成存储与展示双层防护。

详见 `DOCX/功能文档-06-消防新闻模块.md`。

**模块七：数据统计与分析模块**

数据统计：

内容维度：按分类统计知识发布量、浏览量 Top10；

用户维度：统计活跃用户（近 7 天有行为）、各角色用户数；

互动维度：帖子评论 / 点赞数；

**全站运营大盘**：可选时间范围内汇总知识、新闻、器材、论坛与行为浏览等核心指标，管理端卡片展示；

可视化展示：ECharts 绘制柱状图 / 折线图 / 饼图（如分类知识发布量饼图；**折线图支持按日期区间按月趋势**，可与按年浏览趋势并存）；

数据导出：支持将统计结果导出为 Excel（EasyExcel）。
