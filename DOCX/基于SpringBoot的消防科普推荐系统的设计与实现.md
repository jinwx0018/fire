基于SpringBoot的消防科普推荐系统的设计与实现

**模块一：用户管理模块**

用户注册：前端校验普通用户名，手机号格式查重，如果不同的话使用加密算法进行加密存入数据库

用户登录：用户名、密码校验，生成 JWT Token返回Token给前端；

信息管理：用户能够修改头像，手机号，邮箱将参数传给后端进行校验，使用SQL语句更新数据库；

密码找回：支持邮箱链接找回（使用 JavaMail/Spring Mail 发送重置链接）。

**模块二：消防知识内容管理模块**

内容发布：管理员或作者能够更新内容，保存到数据库；

内容查询：支持按分类、标题进行模糊搜索分页查询（HelpPage）；

内容管理：作者、管理员可操作对内容进行增删改查。

内容收藏与分享：用户可收藏知识内容（新增user_collection收藏表），支持生成分享链接。

**模块三：消防知识论坛模块**

帖子发布：用户提交标题、内容，更新数据库 

评论互动：用户评论帖子，记录评论者、帖子 ID 、内容；

点赞功能：Redis Set 集合存储点赞用户 ID（避免重复点赞），点赞数实时更新；

帖子审核：管理员查看待审核帖子，审核通过或者驳回，更新帖子状态（1 - 正常 0 - 待审核 -1 - 违规）以及驳回理由。

**模块四：个性化知识推荐模块**

行为采集：记录用户浏览知识、点赞、评论帖子、收藏内容等行为，更新到用户行为表中；

兴趣分析：统计用户最常访问的分类（Top1）、互动最多的知识标签，查询用户行为表；

推荐计算：优先推荐用户高兴趣分类下的高浏览量；无行为数据时，推荐全站热门知识（浏览量 + 点赞数综合排序）；结果分页返回推荐列表。

智能推荐 - AI 模型接入：在规则推荐基础上，可选接入外部 AI/推荐模型服务（配置 ai.recommend.enabled、api-url、api-key）；调用 AI 接口为每条推荐结果补充推荐理由（recommendReason）或按模型返回顺序重排序，提升可解释性与精准度；未配置或调用失败时自动降级为规则推荐，不影响可用性。

**模块五：消防器材查询**

器材录入：管理员批量导入器材信息到数据库中；

器材查询：支持名称模糊搜索、类型筛选，分页展示；

详情展示：图文展示器材使用步骤、检查要点、常见故障解决。

**模块六：消防新闻模块**

新闻发布：管理员填写标题、内容、地区、紧急等级保存到数据库；

新闻查询：按发布时间、紧急等级排序，支持地区 / 标题搜索，分页查询；

新闻更改：管理员对新闻进行增删改查。

**模块七：数据统计与分析模块**

数据统计：

内容维度：按分类统计知识发布量、浏览量 Top10；

用户维度：统计活跃用户（近 7 天有行为）、各角色用户数；

互动维度：帖子评论 / 点赞数；

可视化展示：ECharts 绘制柱状图 / 折线图 / 饼图（如分类知识发布量饼图、月度浏览量折线图）；

数据导出：支持将统计结果导出为 Excel（EasyExcel）。
