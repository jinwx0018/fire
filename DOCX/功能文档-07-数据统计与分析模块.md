# 功能文档 - 模块七：数据统计与分析模块

本文档对应需求与设计中的**数据统计与分析模块**，说明内容/用户/互动维度统计、**全站运营大盘**、ECharts 图表数据接口（含**区间内按月趋势**）、**EasyExcel 导出**的代码设计与实现。

---

## 一、功能概述

为管理端提供多维度统计：内容维度（**仅已发布**知识 `status=1`，按分类统计发布量、浏览量 Top10）、用户维度（活跃用户、**未删除用户**各角色数）、互动维度（**仅已通过审核**帖子的帖子数、其下属评论数、这些帖子的 **like_count** 合计）；各接口与导出在需要时间筛选时，**`startDate` 与 `endDate`（yyyy-MM-dd）须成对传入或均不传**（未传时：内容/互动/大盘为全时段，**活跃用户仍为近 7 天有行为去重**）。成对传入时区间左闭右开，**跨度不得超过 3650 天（约 10 年）**。

**全站运营大盘**（`GET /admin/statistics/dashboard`）：在相同日期语义下汇总知识、新闻、器材、论坛与行为浏览等核心指标，管理端以卡片展示。

**区间内按月趋势**（`GET /admin/statistics/chart/viewTrendRange`）：与页面日期区间一致，按自然月输出两条序列——**knowledgeViewSum**（已发布知识按**创建月**对 `view_count` 求和，与旧版 viewTrend 口径一致，仅时间范围改为可配置区间）、**behaviorViewCount**（`user_behavior` 中 `behavior_type=VIEW` 按**行为发生月**计数）。**未选日期**时后端按**当年自然年**作为默认区间（与原先「选年 + viewTrend」体验等价）。兼容接口 **viewTrend(year)** 仍保留（按指定自然年按月汇总知识 view_count）。

**EasyExcel** 导出：`type` 为空或 **all** 时除内容/用户/互动外，追加 **运营大盘**各指标与 **月度趋势**行；`type` 为 content/user/interaction 时仅导出对应维度。**viewTrend** 的 **year** 须为四位数字且在合理年份范围内。

**健壮性**：分类/角色聚合行遇非数字 ID 时跳过；导出 IO 失败返回业务错误；非法 **type** 参数拒绝。

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 数据统计 | 同上 + 日期区间 | 内容：**status=1** 且 `deleted=0`，可选 **create_time** 区间；用户：`COUNT(DISTINCT user_id)` SQL（无日期=近 7 天，有日期=区间内）；角色：**deleted=0** 用户 `GROUP BY role_id`；互动：**帖子 `status=1`（已通过）**；评论 **JOIN** 已通过帖；**SUM(like_count)** 仅已通过帖 |
| 运营大盘 | 一屏看多模块 | **dashboard**：知识/新闻/器材条数与浏览合计、**user_behavior** VIEW 总次数、论坛帖子/评论/点赞（与 interaction 口径一致）；时间字段以各实体 **create_time** 为主，左闭右开 **[start, end+1)** |
| 可视化 | ECharts 折线图/饼图（管理端） | categoryPie：[{name, value}]；**viewTrendRange**：双折线（知识创建月 view 汇总 + 行为 VIEW 按月）；**viewTrend**：按年按月仅知识 view 汇总（兼容） |
| 数据导出 | 统计结果导出 Excel | EasyExcel；**全部导出**含大盘与 **viewTrendRange** 月度行；**type** 校验 |

## 四、数据库与数据来源

- **内容维度**：`knowledge_content`，**`deleted=0` 且 `status=1`**；分类聚合与 Top10 均可选按 **create_time** 区间过滤。  
- **用户维度**：`COUNT(DISTINCT user_id)`（**无日期参数**时近 7 天；**有日期**时区间内）；`sys_user` **未删除**（`deleted IS NULL OR 0`）按 **role_id** 聚合。  
- **互动维度**：帖子为 **`forum_post.deleted=0` 且 `status=1`（已通过）**；评论为 **`forum_comment` JOIN 已通过帖**，按评论 **create_time** 区间计数；**`SUM(like_count)`** 仅上述已通过帖（未传日期为全时段）。  
- **大盘**：`knowledge_content`、`news`、`equipment`（均已发布/上架且未删，`status=1`）；新闻/知识 **view_count** 按区间内 **create_time** 筛选后的记录求和；**behaviorViewCount** 为 `user_behavior` 中 `VIEW` 在区间内条数；论坛三项与 **interactionStats** 一致。  
- **图表**：categoryPie 与内容分类统计一致；**viewTrend** 为指定**自然年**内按**创建月份**对 **view_count** 求和（已发布知识），**year** 四位数字校验；**viewTrendRange** 为合法区间内按月合并「知识 SQL + 行为 SQL」结果（受全局日期与跨度限制）。

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | AdminStatisticsController | content、user、interaction、**dashboard**、chart/categoryPie、chart/viewTrend、**chart/viewTrendRange**、export |
| Service | StatisticsService、StatisticsServiceImpl | 注入各 Mapper；**viewTrendRange** 合并按月序列；**dashboard** 聚合多表；export 写响应流 |

### 5.2 统计实现要点

- **contentStats(startDate, endDate)**：  
  - categoryPublishCount：`status=1`、`deleted=0`，可选 **create_time ∈ [start, end+1)**；关联 **knowledge_category**。  
  - viewTop10：同上条件 + **order by view_count desc limit 10**。  
- **userStats(startDate, endDate)**：  
  - activeUserCount：**无日期** → `COUNT(DISTINCT user_id)` 近 7 天；**有日期** → 区间内。  
  - roleCount：**SQL** `GROUP BY role_id` 且排除逻辑删除用户。  
- **interactionStats(startDate, endDate)**：  
  - post：**已通过**（`status=1`）+ **create_time** 条件。  
  - comment：**ForumCommentMapper** 与 **forum_post** 内连接且 **p.status=1**，按评论 **create_time** 区间。  
  - likeCount：**ForumPostMapper** 仅 **status=1** 的 **`sumLikeCountTotal` / `sumLikeCountInRange`**。
- **dashboard(startDate, endDate)**：  
  - 在 **TimeRange** 上与 content 等一致；论坛字段直接复用 **interactionStats** 结果；新闻/器材/知识条数与 view 合计用 **LambdaQueryWrapper** / **QueryWrapper** + **SUM**。

### 5.3 图表数据

- **categoryPie(startDate, endDate)**：与 **contentStats** 分类聚合一致（含日期可选）。  
- **viewTrend(year)**：**已发布**知识，按 **create_time** 落在该年的月份分组 **`SUM(view_count)`**；**year** 经 `resolveTrendYearParam` 校验（四位、1970～明年）。  
- **viewTrendRange(startDate, endDate)**：**KnowledgeContentMapper.selectViewTrendInRange** + **UserBehaviorMapper.selectViewBehaviorCountByMonth**，在 Java 中按 **month** 做 **TreeMap** 合并，缺月补 0；无日期参数时 **start=当年 1 月 1 日、end=次年 1 月 1 日**（左闭右开）。

### 5.4 导出 Excel

- **exportExcel(type, startDate, endDate, response)**：  
  - 设置 response 的 Content-Type、Content-Disposition。  
  - **type** 为空或 **all**：内容 + 用户 + 互动 + **dashboard** 键值行 + **viewTrendRange** 月度行；否则单一维度。非法 **type** 抛业务异常。  
  - **parseTimeRange**：成对日期、最大跨度 **3650** 天；失败时 **BusinessException**；导出 IO 异常同样转业务提示。  
  - 使用 EasyExcel 将 `List<Map<String,String>>` 写入 **response.getOutputStream()**。

## 六、当前实现状态与扩展

- **后端**：`StatisticsServiceImpl` 已实现 contentStats、userStats、interactionStats、**dashboard**、categoryPie、viewTrend、**viewTrendRange**、exportExcel（EasyExcel 写响应流）。  
- **管理端**：`Statistics.vue` — **全站运营大盘**卡片；**日期区间**选择器 +「按区间刷新」；**ECharts** 饼图 + **双折线区间趋势**（调用 **getViewTrendRange**）；**导出**携带当前区间；饼图/趋势加载失败 **ElMessage** 提示。`admin/src/api/statistics.js`：**getDashboard**、**getViewTrendRange**。  
- **管理端「审计日志」**与统计分属不同能力：见《后端接口文档》`GET /admin/audit-logs` 与[功能文档-01-用户管理模块.md](功能文档-01-用户管理模块.md)。
