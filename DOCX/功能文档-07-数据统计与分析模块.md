# 功能文档 - 模块七：数据统计与分析模块

本文档对应需求与设计中的**数据统计与分析模块**，说明内容/用户/互动维度统计、ECharts 图表数据接口与 **EasyExcel 导出**的代码设计与实现。

---

## 一、功能概述

为管理端提供多维度统计：内容维度（按分类统计知识发布量、浏览量 Top10）、用户维度（活跃用户数即近 7 天有行为、各角色用户数）、互动维度（帖子数、评论数、点赞数）；提供图表数据接口（如分类发布量饼图、月度浏览量折线图）供前端 ECharts 渲染；支持将统计结果导出为 Excel（EasyExcel）。

---

## 二、需求与设计要点

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 数据统计 | 内容：分类发布量、浏览量 Top10；用户：活跃用户、各角色数；互动：帖子/评论/点赞数 | 内容：按 category_id 分组 count、按 view_count 排序 limit 10；用户：user_behavior 近 7 天去重 user_id 数、sys_user 按 role_id 分组 count；互动：forum_post/forum_comment 计数、点赞可从 Redis 聚合或 DB like_count 汇总 |
| 可视化 | ECharts 柱状图/折线图/饼图 | 后端提供 JSON 数组接口，如 categoryPie：[{name, value}]；viewTrend：按月份汇总 view_count 或行为量 |
| 数据导出 | 统计结果导出 Excel | 使用 EasyExcel 生成 xlsx，接口返回文件流，Content-Disposition: attachment |

## 四、数据库与数据来源

- **内容维度**：knowledge_content（按 category_id 分组 count；按 view_count 降序 limit 10）。  
- **用户维度**：user_behavior 中 create_time 在近 7 天的 user_id 去重 count；sys_user 按 role_id 分组 count。  
- **互动维度**：forum_post 的 count、forum_comment 的 count、帖子 like_count 之和或 Redis 各帖 Set 的 SCARD 之和。  
- **图表**：categoryPie 来自按分类统计发布量；viewTrend 来自按月份汇总 knowledge_content 的 view_count 或按月份汇总 user_behavior 的浏览行为。

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 类/接口 | 说明 |
|------|---------|------|
| Controller | AdminStatisticsController | content、user、interaction、chart/categoryPie、chart/viewTrend、export |
| Service | StatisticsService、StatisticsServiceImpl | 注入各 Mapper 或自定义 SQL；export 方法接收 HttpServletResponse 写流 |

### 5.2 统计实现要点

- **contentStats()**：  
  - categoryPublishCount：SELECT category_id, COUNT(*) FROM knowledge_content WHERE deleted=0 GROUP BY category_id，关联 knowledge_category 取 name。  
  - viewTop10：SELECT id, title, view_count FROM knowledge_content WHERE deleted=0 AND status=1 ORDER BY view_count DESC LIMIT 10。  
- **userStats()**：  
  - activeUserCount：SELECT COUNT(DISTINCT user_id) FROM user_behavior WHERE create_time >= DATE_SUB(NOW(), INTERVAL 7 DAY)。  
  - roleCount：SELECT role_id, COUNT(*) FROM sys_user GROUP BY role_id，关联 sys_role 取 role_code。  
- **interactionStats()**：  
  - postCount：forum_post 总数（或 deleted=0）；commentCount：forum_comment 总数；likeCount：SUM(forum_post.like_count) 或从 Redis 各 key 的 SCARD 汇总。

### 5.3 图表数据

- **categoryPie()**：返回 List&lt;Map&gt;，每项 name（分类名）、value（发布量），与 contentStats 中分类统计一致。  
- **viewTrend(year)**：按指定年份，按月分组统计 knowledge_content 的 view_count 之和或按 create_time 所在月汇总；返回 List&lt;Map&gt;，如 {month: "2025-01", viewCount: 1000}。

### 5.4 导出 Excel

- **exportExcel(type, startDate, endDate, response)**：  
  - 设置 response 的 Content-Type、Content-Disposition。  
  - 根据 type 查询对应统计数据（如内容统计、用户统计），使用 EasyExcel 的 Write 或注解实体生成 xlsx，写入 response.getOutputStream()。  
  - startDate、endDate 可限制 create_time 或行为时间范围。

## 六、当前实现状态与扩展

- 当前 StatisticsServiceImpl 中 contentStats、userStats、interactionStats、categoryPie、viewTrend、exportExcel 可能为占位实现（返回空列表或空 Map），需按上述 SQL 与 EasyExcel 补全。  
- 前端 ECharts 调用 chart 接口后，将返回的数组绑定到 series.data 即可绘制饼图、折线图等。
