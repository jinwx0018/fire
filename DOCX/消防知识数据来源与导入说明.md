# 消防知识数据来源与导入说明

本文档说明如何为「消防科普推荐系统」获取并导入足够的消防知识数据，供毕设演示与推荐效果使用。

---

## 一、数据库表结构简要说明

- **knowledge_category**：分类（如火灾预防、逃生自救、器材使用等）
- **knowledge_content**：知识正文，字段包括 title、category_id、content、cover、summary、author_id、view_count、like_count、status、deleted 等
- **knowledge_tag** / **knowledge_content_tag**：标签及与内容的关联（可选）

内容需先有分类（category_id），且 author_id 需为已存在的用户 ID（如管理员账号）。

---

## 二、推荐数据来源（合规、可引用）

### 1. 官方与政府公开资源（优先推荐）

| 来源 | 说明 | 获取方式 |
|------|------|----------|
| **中国消防网 / 应急管理部** | 应急管理部、消防救援局官方信息 | 打开官网栏目（如科普教育、消防安全），复制整理文字，注明出处 |
| **各地消防救援总队/支队官网** | 各省市消防官方科普文章 | 同上，按「火灾预防」「逃生自救」「器材使用」等分类整理 |
| **全国消防科普教育基地** | 权威科普内容 | 官网或合作平台发布的图文，可整理为一条条知识 |
| **国家应急广播** | 应急与消防科普 | 网站/公众号文章可整理后录入，注明来源 |

使用建议：**仅做学习与毕设演示**，手动或半自动整理为「标题 + 摘要 + 正文」，注明“来源：xxx”，避免商业使用与二次传播侵权。

### 2. 开放数据与数据集

- **各大学 / 科研机构开放数据**：部分智慧消防、公共安全数据集包含“消防知识”“安全规范”类文本，可检索「消防 数据集」「fire safety dataset」等，按需选用并遵守其许可协议。
- **百科类**：维基百科、百度百科等有消防相关词条，可作参考与摘要编写依据，**不要整篇照抄**，应改写、缩写并标注参考来源。

### 3. 自行编写与整理

- 根据 **GB 等国家标准**（如建筑防火、灭火器配置）整理成通俗的「知识点」条目。
- 将教材、培训资料中的要点改写成短篇科普（标题 + 摘要 + 正文），用于充实分类与推荐效果。

## 三、数据入库的几种方式

### 方式一：管理端后台手动录入（适合少量）

1. 先登录管理端，在「分类管理」中新增分类（如：火灾预防、逃生自救、灭火器使用、家庭防火等）。
2. 在「知识内容」中逐条新增：选择分类、填写标题、摘要、正文、可选封面图，保存发布。
3. 每条知识会自动带 author_id（当前登录用户）。

### 方式二：执行种子数据 SQL（适合快速建库、演示）

项目已提供 **DB/seed_knowledge_sample.sql**（若已创建），在 MySQL 中执行即可插入示例分类与若干条消防知识。

- 执行前请确保：
  - 已执行 `01_sys_user.sql`、`02_knowledge.sql`；
  - 数据库中已存在至少一个用户（如管理员），且其 `id` 与 SQL 中的 `author_id` 一致（通常为 1；若不同请先修改 SQL 中的 author_id 再执行）。

### 方式三：批量导入脚本（适合大批量）

- 将数据整理为 JSON 或 Excel（每行：标题、分类名或 category_id、正文、摘要、封面 URL 等）。
- 写本地脚本（如 Node/ Python）读取文件，循环调用后端 **POST /api/content** 接口（需先登录拿到 token，请求头带 `Authorization: Bearer <token>`），即可批量写入。
- 分类可先通过管理端或 **GET/POST /api/admin/content/category** 维护好，脚本中按分类名映射到 category_id 再提交内容。

### 方式四：从网页采集（需谨慎合规）

- 仅针对**明确允许转载或开放授权**的站点，且仅用于毕设与学习。
- 可用爬虫抓取列表页 + 详情页，抽取标题、正文、摘要，再通过方式二或方式三入库；务必遵守 robots.txt、版权声明与网站条款，并注明来源。

## 四、建议的数据规模与分类示例

- **分类建议**：火灾预防、逃生自救、灭火器使用、家庭防火、公共场所防火、电动车/电气防火、消防法规与常识等（约 6～10 个）。
- **数量建议**：每条分类下至少 5～15 条知识，总数 50～150 条即可支撑推荐与冷启动效果；毕设演示可先做 30～50 条，再逐步补充。
- **字段建议**：标题简洁明确；摘要 1～2 句；正文可富文本或纯文本；封面可为空或使用免费图库/占位图。
