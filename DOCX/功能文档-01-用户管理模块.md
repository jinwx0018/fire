   # 功能文档 - 模块一：用户管理模块

本文档对应《毕业设计需求文档》《基于SpringBoot的消防科普推荐系统的设计与实现》中的**用户管理模块**，说明需求、接口、数据库与**代码设计与实现**。

---

## 一、功能概述

用户管理模块负责用户注册、登录、个人信息维护与密码找回，并为管理端提供用户列表查询。登录采用 JWT 无状态认证，密码使用 BCrypt 加密存储。

---

## 二、需求与设计要点（来源：需求文档 + 设计与实现）

| 功能点 | 需求描述 | 设计实现要点 |
|--------|----------|--------------|
| 用户注册 | 前端校验用户名、手机号格式与查重；密码加密入库 | 后端再次校验用户名/手机号唯一性；使用 BCrypt 加密密码；默认角色为普通用户(role_id=1) |
| 用户登录 | 用户名、密码校验，生成 JWT Token 返回前端 | 校验用户存在、密码匹配、账号未禁用；调用 JwtUtil 生成 Token，将 userId、username、role、avatar 等返回 |
| 信息管理 | 用户可修改头像、手机号、邮箱；参数传后端校验后更新数据库 | 仅允许修改当前登录用户；手机号、邮箱修改时校验不与他人重复；使用 MyBatis-Plus 按 ID 更新 |
| 密码找回 | 支持邮箱链接找回 | 接口：sendResetEmail、resetByToken；使用 JavaMail 发邮件并读写 sys_password_reset_token 表 |

## 三、接口列表（对应《后端接口文档》）

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /user/register | 用户注册 | 否 |

## 四、数据库设计

脚本位置：`DB/01_sys_user.sql`。

| 表名 | 说明 |
|------|------|
| sys_role | 角色表，角色编码 USER/AUTHOR/ADMIN |
| sys_user | 用户表，含用户名、密码(BCrypt)、手机、邮箱、头像、角色、状态 |
| sys_password_reset_token | 邮箱找回密码用 Token（过期时间、是否已使用） |

**sys_user 关键字段：** id, username(唯一), password(128 位存 BCrypt), phone(唯一), email(唯一), avatar, role_id(外键 sys_role.id), status(0 禁用 1 正常), create_time, update_time。

## 五、代码设计与实现

### 5.1 包与类结构

| 层次 | 路径 | 说明 |
|------|------|------|
| Controller | controller/UserController.java | 用户端：注册、登录、信息、密码相关接口 |
| Controller | controller/admin/AdminUserController.java | 管理端：用户列表分页 |
| Service | service/UserService.java、impl/UserServiceImpl.java | 注册、登录、信息维护、密码找回、管理员列表 |
| Mapper | mapper/SysUserMapper.java、SysRoleMapper.java | MyBatis-Plus BaseMapper，对应 sys_user、sys_role |
| Entity | entity/SysUser.java、SysRole.java | 与表结构一致，驼峰映射 |
| DTO | dto/RegisterReq.java、LoginReq.java、LoginVO.java、UserInfoVO.java | 注册/登录请求、登录响应、用户信息响应 |
| 工具/配置 | util/JwtUtil.java、config/SecurityConfig.java | JWT 生成/解析/校验；PasswordEncoder(BCrypt) Bean |
| 拦截器 | interceptor/JwtInterceptor.java、config/WebMvcConfig.java | 请求头 Authorization: Bearer &lt;token&gt; 解析，将 userId 放入 request；放行注册/登录/密码找回路径 |

### 5.2 认证流程设计

1. **放行路径**：WebMvcConfig 中 excludePathPatterns 包含 `/user/register`、`/user/login`、`/user/password/*`，这些请求不经过 JwtInterceptor。
2. **需认证请求**：请求头携带 `Authorization: Bearer <token>`，JwtInterceptor 截取 token，调用 JwtUtil.validate 与 parse，将 userId 写入 `request.setAttribute("userId", userId)`，后续 Controller 从 request 取 userId。
3. **Token 生成**：登录成功后 JwtUtil.generate(userId, username, role) 使用 HS256、配置的 secret 与过期时间生成 JWT，subject 存 userId，claim 存 username、role。

### 5.3 注册实现要点

- **UserServiceImpl.register(RegisterReq)**：  
  - 使用 LambdaQueryWrapper 按 username、phone 分别查询，若已存在则抛出 BusinessException("用户名已存在") 或 "手机号已注册"。  
  - 新建 SysUser，password 使用 `passwordEncoder.encode(req.getPassword())` 写入。  
  - roleId 设为 1（普通用户），status 设为 1，执行 userMapper.insert(user)，返回 user.getId()。

### 5.4 登录实现要点

- **UserServiceImpl.login(LoginReq)**：  
  - 按 username 查询 SysUser；若为空或 `!passwordEncoder.matches(req.getPassword(), user.getPassword())` 则抛 "用户名或密码错误"。  
  - 若 user.getStatus() != 1 抛 "账号已被禁用"。  
  - 根据 user.getRoleId() 查 SysRole 取 roleCode；调用 jwtUtil.generate(...) 得到 token。  
  - 构造 LoginVO（token、userId、username、role、avatar）返回。

### 5.5 信息管理实现要点

- **getInfo(userId)**：根据 userId 查 SysUser，若为空抛 NOT_FOUND；复制到 UserInfoVO，并查角色得到 roleCode 填入。  
- **updateInfo(userId, avatar, phone, email)**：查当前用户；若传入 phone，先查是否被其他用户占用（排除当前 userId），邮箱同理；有则抛 "手机号已被使用" 等；最后 set 字段并 userMapper.updateById(user)。

### 5.6 管理员用户列表

- **adminUserList(pageNum, pageSize, username, role)**：使用 MyBatis-Plus 的 Page 与 LambdaQueryWrapper，按 username 模糊、role（需关联 sys_role 或 role_id）筛选，selectPage 后将每条 SysUser 转为 UserInfoVO（含角色码），返回 IPage&lt;UserInfoVO&gt;。

## 六、扩展与待实现

- 密码找回：sendResetEmail、resetByToken 需配置 JavaMail（Spring Mail）发邮件并读写 sys_password_reset_token 表；未配置时重置链接仅在服务端日志输出。  
- 角色权限：管理员列表等可结合角色码在拦截器或 AOP 中做权限控制（当前仅做登录态校验）。
